# Echoes of the Lost Kingdom — メインゲームデザイン

> 本書は企画の原案を定める。やりたいことを見失わないための記録。

---

## 1. コンセプト

### 1.1 一行ピッチ
**戦略が結末を決め、感情が周回を生む ローグライトSLG** 。亡国の王女の声に呼ばれ、千年前の過去で帝国の侵攻と戦う。1ラン2〜3時間を複数周行うことで、最終的に望む結末にたどり着くことを目指す。

### 1.2 ジャンル定義
本作のジャンルは **「国盗りSLG × ローグライト」** の2軸融合がベース。そこに**ストーリー**を加えて間口を広げつつ、**オートバトル**によってゲームとして成立させる。

- **国盗りSLG要素**: マップ上で自領と敵領を奪い合う領土戦略や内政要素を楽しむSLG。1プレイが長くなりがち
- **ローグライト要素**: 1プレイ（ラン）を反復し、ラン外でメタ進行を蓄積する構造。1ランは短い前提
- **ストーリー要素**: 選択肢ではなく、戦略判断が結末を決め、その結末を変えるために周回する。ローグライト*SLGの部分と密接に連動する
- **オートバトル要素**: 戦闘そのものは自動進行で、プレイ時間を短縮する。戦闘の結果は事前の編成や配置で決まる

> **市場分析**：特に「**ストーリー付きのSLG**」は、熱量の高いコアファンを持つ一方で、新作・続編の頻度は限定的で、常に供給不足の領域であると考える。

### 1.3 ターゲット層
**「重厚な国盗りシミュレーションゲームを、もう少し気軽に遊びたい層」** を主要ターゲットとしつつ、**「ストーリー付きのSLG/SRPGが好きなライト層」**の取り込みも狙う。

具体的には以下のような層を想定する：

- 『信長の野望』『Civilization』などの長大戦略SLGを愛好していたが、プレイ時間が取れずに遊ぶ頻度が減ってきた層
- 『Into the Breach』『FTL: Faster Than Light』のような、ローグライトの中でもSLG要素が強い重めのタイトルが好きな層
- 『ファイアーエムブレム』『タクティクスオウガ』など、戦略/戦術要素のあるストーリーゲーム（J-SRPG）が好きな層

上記以外の近縁/参考タイトル：
- 『ユニコーンオーバーロード』：独自色の強いSLGパートと王道ファンタジー戦記の組み合わせ。本作のコンセプトの一つである「SLGとストーリーの融合」には根強い需要があると示す好例。
- 『ブリガンダイン ルーナジア戦記』：ライト寄りの国盗りゲームのイメージとして。戦略/内政面はシンプルで、戦術要素としてのSRPGパートがゲーム体験の中心。
- 『The Last Spell』：ローグライト×重厚なSLGの成功例。一戦闘が重すぎるという難点を本作ではオートバトルで解消し、より戦略面に特化したプレイ感のゲームを目指す。
- 『Inscryption』：ローグライトというジャンル自体をメタ的にストーリーに組み込んだ成功例。
- 『Against the Storm』：別ジャンルだが、1プレイの長い「街づくり」ゲームを軽いローグライトに落とし込んだ成功例。

『Hades』のような軽めのストーリーつきローグライトとは明確に差別化し、「1回のランに2〜3時間集中して、部隊・資源・戦線を管理する重厚なSLG」を志向する。

一方で、SLGパートでの判断がキャラクターの結末に影響するという物語フックによって、戦略ゲームに馴染みの薄いライト層にも訴求する。

### 1.4 USP（独自性）

本作の独自性は「**戦略が結末を決め、感情が周回を生むローグライトSLG**」の一点に集約される。これは「**物語とゲーム性のシンクロ**」の双方向ループであり、本作の構造的核である。

**双方向シンクロ**：
- **ラン内（ゲーム → 物語）**：多戦線でのリソース管理という戦略判断が、誰を救うか／誰を見捨てるかという物語上の選択になる。「味方を救うために防衛線力を他の戦線に回すべきか」といったプレイヤーの判断の積み重ねによって結末が決まる。
- **ラン外（物語 → ゲーム）**：王女を救えなかった後悔や、次はこのキャラクターを救いたいというプレイヤーの感情が、次の周回への動機を生む。（メタ強化という報酬だけでなく、物語から来る感情が周回の動機となる）

### 1.5 想定プレイ時間

| 単位 | 時間目安 |
|---|---|
| 1ラン | 約2〜3時間 |
| ノーマルエンド到達 | 約3〜4ラン（合計約8〜12時間） |
| トゥルーエンド到達 | 約10ラン（合計約25〜30時間） |
| クリア後のやり込み | 上記＋α |

トゥルーエンド到達までの目安総時間は **約30時間**。商業作品としては中規模、ローグライトとしてはやや大型。

## 2. 物語と世界観

奇をてらわず、多くのプレイヤーにとって馴染みやすいファンタジーループものの王道プロットとする。

### 2.1 世界観の概要
本作の舞台は架空のファンタジー世界。プレイヤーは **現代と約1000年前の過去** という、2つの時代を往復する。

- **現代パート（拠点）**: 亡国の遺跡があるだけ。世界観を深くは明かさない
- **過去パート（ラン）**: 中世ファンタジーの時代。剣と魔法、騎士と帝国の世界

### 2.2 プロローグ
遺跡探検を趣味とする主人公（現代の一般人）は、ある日、過去に滅んだとされる王国の遺跡を訪れる。

そこで主人公は、不思議な声に呼ばれ、過去の世界へと飛ばされる。

声の主は **亡国の王女**。今まさに大帝国の侵攻によって滅ぼうとしている王国を救うべく、最後の希望として禁術「異世界からの英雄召喚」を行使したのだった。

しかし召喚されたのは、英雄でも何でもない一般人の主人公。それでも王女の期待に応えようと、主人公は王国軍を指揮して帝国と戦うことを決意する——。

> ゲームシステム上、初回ランは敗北前提の難易度設定とする。物語的にも、初回ランは「奇跡が起きる物語」ではなく「ありえなかった希望が砕かれる物語」として演出する。

### 2.3 キャラクター案

#### 主人公（プレイヤー）
- 現代に生きる一般人。遺跡探検が趣味
- 性格などはプレイヤーの想像に委ねるため、明確なキャラ立ては行わず、セリフを発さない（無口プロタゴニスト）
- 英雄ではないが、王女の期待に応えようとする人物

#### 王女（エリシア（仮））
- 10代後半。王女として、最前線で軍を指揮する。聖剣の力で異世界（千年後の未来）から主人公を召喚した。
- **公の顔**: 勇敢で、先陣を切って敵と戦い、味方を鼓舞する凛とした姿
- **裏の顔**: 本心では恐怖と絶望に押しつぶされそうな普通の少女。ふとした拍子に主人公にその姿を知られ、弱音を見せるようになる
- プレイヤーにとって「救いたい」という感情が生まれるような、王としての器と魅力的なヒロイン像を兼ね備えた人物として描く必要がある

#### 固有ユニット（5名くらい）

SRPGのような数十単位の固有ユニットは登場させない。数人のキャラクターに対して「避けたい悲劇」や「王女や他のキャラクターとの関係」の深掘りを行い、プレイヤーにとって「救いたい」小目標となるように設計する。

- 「あるラウンドまでに救援をしなければ殺されてしまう」といったような、プレイヤーの戦略判断によって救える悲劇をそれぞれに準備
- 救援に成功するとそのランの間だけ仲間になるが、仲間になった後でイベントをこなすなど条件を満たすと、仲間状態が永続化。それ以降のランでは初期から仲間になる。（周回の小目標として設計）

#### 皇太子
- 帝国の皇太子。本作で主にプレイヤーが対峙する敵将で、ノーマルエンドでのボス。
- **軍略の天才**。占領地での容赦ない略奪と殺戮で恐れられる、表面上は **明確な悪役**
- 本質は正々堂々と戦う戦士であるが、闇の力で操られていた。真の黒幕がトゥルーエンドのボス。

#### 真の黒幕
- トゥルーエンドでのラスボス。詳細未定。

### 2.4 エンディング構造

本作は **3つの主要エンディング** を持ち、それぞれに到達条件と物語の決着が明確に定義される。

| エンディング | 達成条件（概略） | 物語の結末 |
|---|---|---|
| **バッドエンド** | 規程ラウンドまでに本拠地が陥落、または皇太子に敗北 | 敗北寸前、王女が禁術で主人公を元の世界に送り返す。涙ながらに笑顔で「ありがとう」と告げる |
| **ノーマルエンド** | 規程ラウンドまで本拠地を防衛し、ラスボスの皇太子を撃退する | 皇太子は撤退し、帝国と停戦。主人公は未来へ帰還するが、現代の遺跡は廃墟のまま——結局王国は滅亡していたことを知る |
| **トゥルーエンド** | 王女との仲を深めるイベントをこなすなど、条件を満たす | 真の黒幕を聖剣の力で倒すが、代償として主人公は帰れなくなる。王女と結ばれて王国で生涯を過ごすことに |

#### バッドエンドの特殊性
バッドエンドは「ゲームオーバー」ではなく、**物語のループを駆動する装置** として機能する。

王女が主人公を責めるどころか「ありがとう」と感謝して送り返すことで、プレイヤーには「努力は無駄ではなかった」という肯定感と、「彼女の期待にもっと応えたい」という再挑戦の動機が同時に与えられる。

これは本作におけるローグライトの心理設計の核である。「ゲームオーバー＝挫折」ではなく「大切な人に救われる罪悪感」へと感情の質を変換する。

#### 1周目の必敗構造
**1周目は構造的にバッドエンド確定** とする。防衛に成功しても最終ラウンドにラスボスの皇太子が本拠地に攻め入り、これは2週目以降に特定イベントを見ないと確定負けイベ。

#### ノーマルエンド
ノーマルエンドは表面上「クリア」だが、未来で「結局国は滅んでいた」という真実を突きつける **ビターエンド** である。主人公が真の黒幕の存在を認知することで、次の周回への決意を新たにする。

#### トゥルーエンド達成の条件
詳細未定だが、王女との仲を深めるイベント、固有ユニットの救出、強敵の撃破などの複合的な条件を達成する必要がある作りで、ノーマルエンド達成よりも多くの周回数を要求する想定。

- 周回ごとに固有ユニットを永続化する、謎が部分的に明らかになる、などの小目標/小報酬を段階的に設けることで、飽きずに周回できるような作りが求められる。

### 2.5 周回前提のシナリオ設計
本作のシナリオは **「周回を重ねることで真相と希望に近づく」** という構造を意図的に持たせる。「クリア後のおまけ周回」ではなく、**周回そのものが物語の本編** である。

#### 設計意図
- シナリオが **ゲームシステム（ローグライト周回）と完全に同期** している
- プレイヤーが「攻略のためにシナリオを進める」のではなく「シナリオを知るために周回する」状態になることを目指す

#### イベント解禁とメタ進行の連動
ラン内でのイベントは、周回によるメタ強化の進行度に応じて解禁される。これにより、どれだけ効率的にプレイしてもトゥルーエンドには一定回数の周回が必要な構造を担保する。

#### 完全クリア後の扱い
トゥルーエンド達成は **ゲームの完全な終わり** として扱う。再度ストーリーを遊びたい場合は **クリア前のセーブデータからやり直す** 必要がある。

- 設計判断として、物語的に「ループが完結した」状態を表現する。とはいえ、クリア後のやり込み要素は物語とは別と割り切って実装することも検討。

## 3. プレイ体験

### 3.1 ゲームの 4 階層構造
ゲームは以下の4階層で構成される：

```
【メタ層（拠点）】
  現代の王国遺跡で「王国の記憶」を消費してメタ強化・イベントの開放
        │
        ▼ 
【ラン層（1ラン約2〜3時間）】
  過去の世界での防衛作戦。12ラウンド～を想定。
        │
        ▼
【ラウンド層（1ラウンド）】
  内政フェーズ → 師団配置フェーズ → 戦闘解決フェーズ
        │
        ▼
【戦闘層】
  最大6体のオートバトル
```

### 3.2 コアループ（拠点⇄ラン）
ゲームは **「拠点（現代の王国遺跡）」と「1ラン（過去の世界での防衛作戦）」の往復** で構成される。

```
┌─────────────────────────────────────────────────────────┐
│ 【拠点：現代の王国遺跡】                                │
│   ・物語進行                                            │
│   ・「王国の記憶」を消費してメタ強化                    │
│   ・遺跡の段階的復旧 → 新要素アンロック                 │
│   ・固有ユニットの永続育成                              │
└─────────────────────────────────────────────────────────┘
                       │ ラン開始
                       ▼
┌─────────────────────────────────────────────────────────┐
│ 【1ラン：過去の世界での防衛作戦】（約2〜3時間）        │
│   各ラウンド：内政 → 配置 → 戦闘解決                      │
│                                                         │
│   エンド判定：バッド／ノーマル／トゥルー                │
└─────────────────────────────────────────────────────────┘
                       │ ラン終了
                       ▼
              「王国の記憶」を持ち帰る → 拠点強化 → 次ランへ
```

### 3.3 プレイヤーが取る主な判断（3 スケールの並列意思決定）
プレイヤーは以下の **3つの時間スケールの判断** を並行してこなすことを楽しむ。

1. **ラウンド単位の判断**：「このラウンドで行動力や資源を何に使うか／どの戦線にどのユニットを置くか」
2. **ラン単位の判断**：「どのユニットを育成するか／どの戦線に注力するか／味方救援や強敵撃破を狙うか防衛で凌ぐか」
3. **ラン横断の判断**：「持ち帰った王国の記憶を何に投資するか／次のランで何を試すか」

この **「3スケールの並列意思決定」** が、本作が既存のローグライトとも国盗りSLGとも異なる遊び体験を提供する核となる。

### 3.4 1ランの体験フロー
1ランは 12ラウンド（仮） で構成される。各ラウンドは以下の3フェーズで進行する：

1. **内政フェーズ**：行動力を消費して各種アクションを選択
2. **師団配置フェーズ**：マップ上の自領・敵領へ師団を配置
3. **戦闘解決フェーズ**：発生した全戦線を順次自動戦闘

### 3.5 コアの遊び体験
- **「2～3時間集中して戦略を組み立てるプレイ体験」**
- 戦闘はオートバトルだが、戦闘前の編成・配置・強化が結果を決める
- 周回ごとに「持ち帰った記憶で何を強化するか」のメタ判断
- 物語進行と周回が一体化し、「シナリオを知るために周回する」状態になる

## 4. 戦術：配置と戦闘

### 4.1 戦闘システム原則
詳細はプロトタイプや試遊版を作って遊びながら変更する想定なので、大方針を定める。

- **ラン構造がゲームの本体**：複数戦線への配分判断がゲームの面白さの基盤。戦闘はプレイヤーの戦略判断が正しいかの答え合わせに過ぎない。よって戦闘自体に過度な複雑さは要らない。
- **わかりやすいこと**：ルール説明はゲーム画面で完結できる程度のわかりやすさ。戦闘を見ればプレイヤーが「なぜ負けたか」が理解できる。非公開情報の禁止。
- **予測可能であること**：過度なランダム要素の禁止（セーブロードによる戦闘のやり直しは推奨しない）。本作のランダム性は、敵の編成や戦線の配分、ユニット獲得の部分にとどめ、戦闘は始まった時点で結果が決まっているべき。
- **複数の勝ち筋**：ローグライト＝ユニット獲得はランダムなので、「持ち物検査」になってはいけない。特定のユニット相性に依存せず、様々な勝ち筋が成立する必要がある。
- **シナジーベースのデザイン**：オートバトラーの常道。同じ属性のユニットを集めることによるシナジー効果は、初心者プレイヤーにピックの道筋を示し、「様々な勝ち筋」を成立させるカギにもなる。
- **高い床**：ユニット獲得がランダムなので、シナジー前提の極端な性能のユニットやハズレ枠はなるべく作らない。シナジーなどの強化効果は加算ベースにし、「なくても戦えるがあれば嬉しい」デザインとする。
- **緩い天井**：逆に理想的なシナジーが組めたときの強さは青天井に近くてよい。プレイヤーの快感になるような要素は過剰に取り上げない。複数戦線を抱えるゲーム性のため、1つの最強編成が組めてもすべてを解決することはできない。

### 4.2 師団編成と配置ルール
- 師団は **最大6体** のユニットで構成される1戦線分の戦闘単位
- ユニットは必ずしも常に6体揃える必要はなく、空き枠のある師団も配置できる

#### 配置の仕組み
配置フェーズで、プレイヤーはマップ上の各領地にユニットを割り当てる：

- **自領への配置**：防衛師団として機能する。敵が侵攻してきた場合に戦闘に参加する
- **敵領への配置**：侵攻師団として機能する。戦闘解決フェーズで攻め込む

配置されなかったユニットは待機となり、いずれの戦闘にも参加しない。

#### 配置の引き継ぎ
前ラウンドに配置したユニットは、翌ラウンドもそのまま同じ領地に残る。毎ラウンドゼロから組み直す必要はなく、欠員が生じた枠だけを差し替えながら運用することで操作負担を軽減する。

### 4.3 戦闘の勝敗判定（段階評価・戦況ゲージ）
- 戦闘には一定の「ターン制限」を設け、ターン制限内の決着状況と「戦況ゲージ」（彼我の損耗差に基づく戦況の傾き）によって、結果を評価する。
- たとえば侵攻戦で敵を全滅させれば1回で占領できるが、時間切れでの優勢勝利であれば2回必要、のようなバランスで検討。

### 4.4 戦闘AIロジック
オートバトルの核。わかりやすく予測可能でありつつ、面白いことが求められる。様々な案が考えられるため、試遊してみての判断。

- たとえば各ユニットごとに「最も柔らかい敵を狙う」などのターゲッティングルールを個別に定めると、何が起こっているかわかりにくい。
- 逆に統一的な行動ロジックを決めると、戦術やユニットデザインが単純化しすぎる懸念もある。

### 4.5 ターン進行構造
- ユニットの行動順は素早さ SPD の降順
- 行動完了のたびに未行動リストを動的に再計算することで、行動中に発生したバフ・デバフの効果を反映する
- SPD が同値の場合、特定の優先度で決定する（ランダムにはしない）。たとえば、陣営順（味方 ＞ 敵）で、その中では配置順。

### 4.6 ダメージ計算
- HP=兵力とみなし、原則的に残HPが高いほどダメージが大きくなる設計方針を採用する。これにより、倒しきれなくても削ることに意味が生まれるため、少しの数値差で戦闘結果が極端に変わることが抑制できると期待。
- 攻撃力は ATK 防御力は DEF で、シンプルに。物理/魔法の区別や属性相性などは採用しない。DEF は減算ではなく除算で扱うことで、高すぎる防御でダメージが通らない構造を防ぐ。

### 4.7 観戦モード／速度設定／プレイヤー介入／オートモード

#### 観戦モードと速度設定
戦闘は **じっくり挙動を観察できる** ことと、高速周回のために **最高速度で流し見るかスキップされる** ことの両方に耐えられるように設計する。

| モード | 概要 |
|---|---|
| ステップ送り | クリックで1アクションずつ確認 |
| 等倍 | 通常速度で自動観戦 |
| 2倍速 | 2倍速で自動観戦 |
| 4倍速 | 4倍速で自動観戦 |
| スキップ | 戦闘アニメをスキップし、結果のみを即時表示 |

#### 複数戦線の観戦
同ラウンドに複数の戦闘が発生した場合、1戦ずつ解決する（解決順序は別途検討）。そのラウンドの戦闘すべてをまとめてスキップして結果サマリだけ見ることもできる。

### 4.8 ユニットロストの非採用
本作では **ユニットのロスト（永久喪失）は採用しない** （固有ユニットのイベントによるロストを除く）。

- ロストがあると、プレイヤーは「重要ユニットを失わないために毎戦闘を注視し、危なければ撤退する」必要に迫られる。これはスキップ・高速観戦の価値を損ない、「1ランの時間を抑える」という大方針と衝突する。
- 戦闘不能ユニットを出した場合のペナルティについては、次ラウンドで一時的に使用不可にする、回復にコストを要求する、などを検討。

### 4.10 検討中のアイデア

#### プレイヤー介入
通常の観戦中、すべてのユニットが行動してターンが終了するたびにプレイヤーの介入タイミングを設ける。以下のアクションが行える想定。

- **アクティブスキルの発動や消費アイテムの使用**
- **撤退**　→勝てそうにない場合に自軍の損害を抑えるため

#### リーダースキル
**師団のリーダーが発動する固有のアクティブスキル** 

- 各師団ごとにリーダーユニットを1体指定する。固有ユニットや一部のレアユニットだけリーダースキルを持ち、プレイヤー介入タイミングで使用できる。
- ゲージ制でここぞというときにだけ使えるか、消費アイテムのようなものが必要か、といったイメージ。

#### 撤退
偵察しなければ敵編成が分からない想定のため、戦ってみて初めて負けそうだと判断するケースは残る。撤退は、戦闘不能に何らかのデメリットがある場合の損切り手段として機能する。

#### 奇襲
スキル等により、戦闘の前に奇襲判定を行い、奇襲が成功した場合は敵の陣形を乱したりデバフを与えるなどのマイナス効果を想定。奇襲を防ぐスキルもあり。

## 5. 戦略：采配と領土

### 5.1 マップと領土システム
マップは **王国全土を表す固定マップ** とする。

#### 領土の状態
領土の状態は **自領 / 敵領の2種のみ** 
- **自領**：毎ラウンドの収入源となる。自軍師団を配置して防衛・出撃の拠点とする
- **敵領**：すべての敵領に敵軍が駐留している。自軍師団を配置して攻め込むことで占領できる

#### ラン開始時の状態
ラン開始時、マップはすでに敵に大半を占領された状態からスタートする。初期の自領数は少なく、プレイヤーは劣勢の状態から戦いを始める。

### 5.2 占領・防衛ルール
戦闘の勝敗（完勝/辛勝）に応じて占領ポイントを得て、規定値に達すると占領できるイメージ。

#### 戦闘の損耗の引き継ぎ
- 勝敗にかかわらず、戦闘で受けたダメージは次ラウンドに引き継がれる
- HPが0になったユニットはロストせず **負傷状態** となる

### 5.3 戦線数・収入・反攻イベント

#### 戦線数
戦線は **自領と敵領が隣接しているタイル境界** から自動的に発生する。マップの占領状況に応じて常時 **2〜4戦線** 程度が自然に生まれる状態を想定する。

#### 収入
毎ラウンド終了時に、原則として **自領数 × 収入単価** の**ゴールド**を獲得する。

#### 全領地奪還の禁止
- 「全土制圧による早期決着」は成立しない想定。

### 5.4 1ラウンドの流れ（内政→配置→戦闘→ラウンド終了）
1ラウンドは以下の3フェーズで構成される：

```
内政フェーズ → 師団配置フェーズ → 戦闘解決フェーズ
```

#### 戦闘解決フェーズの処理順
同一ラウンドに複数の戦闘が発生した場合の順序は要検討。自軍の侵攻と防衛はフェーズをわけるのが処理しやすいか。

#### 侵攻の制限
ランの進行ペースをある程度コントロールするため、1ラウンドに自軍が侵攻できる敵領の数に制限を設けることも検討。

#### 戦闘の報酬
- 戦闘ごとに勝敗に応じて報酬を得るか、敵を撃破するごとにスコアが貯まるような形式か。報酬としては、ゴールドなどだけか、ユニットなどのドロップも含むのか。

### 5.5 内政システム

#### ラン内資源体系
何をするにもゴールドが必要な想定。建築には建築資材が必要など、複数の資源を扱うかは検討。

#### 行動力
毎ラウンド開始時に **全回復** し、繰り越しはない。行動力はカツカツに設計されており、常に取捨選択が求められる。メタ強化によって上限を解禁できる。

#### 内政コマンド一覧
偵察（敵戦力の事前把握）、招集（新規ユニットのドラフト）、建設（自領の強化）、ユニット強化、イベントなど

- 偵察はや建設はユニットを指定して実行させることも検討。内政能力を設ける。内政に使ったユニットはそのラウンドの戦闘には出せない。

##### 5.5.1 建設
各領地に建設枠が1-2つあるようなイメージ。建設する建物により、ゴールド収入や経験値の増加、防衛戦でのバフ効果などが得られる。

##### 5.5.2 偵察
敵領地を攻める前に構成を確認したり、このラウンドに攻めてくるかなどを事前に知れる想定。

##### 5.5.3 招集
ユニットドラフト。このコマンドでユニットを増やすか、戦闘の報酬か、両方か。内政能力が高ければレアユニットがでやすいなど。

##### 5.5.4 治療
ゴールドを消費してHPを回復＝兵を補充する想定。行動力は使わない。

### 5.6 ユニット強化（ラン内・戦闘間の成長）
ラン内でのユニット成長は、原則ランごとにリセットされる。

#### 5.6.1 レベルアップ（EXPアイテム・Lv1-5）
- 各ユニットは「EXPアイテム」を消費することでレベルを上げることができ、レベルごとに強化項目を選択できるイメージ。
- EXPアイテムは基本的に戦闘で入手する。戦闘で直接経験値を得る形式にしないのは、終盤加入のユニットのレベル上げが大変だから。

#### 5.6.2 装備
- ユニットのベースデータに対し、戦闘外で「装備品」を付与することができる
- **効果**：新しい技の追加、ステータス補正など
- **制限と着脱**：各ユニットが装備できる数には制限を設ける。戦闘外であればいつでも自由に付け外しが可能な想定
- **ラン内完結**：ユニットのレベルと同様、装備獲得もラン内で完結し、ラン開始時にリセットされる

## 6. 周回：ローグライト構造

### 6.1 敵強度スケール
本作の周回による強化要素（拠点強化）は、原則的に **上限解禁型** であり、ランの開始時から全軍が強化されるような設計ではない。そのため、 **単に周回回数によって敵が強化されることはない**想定

- なお、同一ラン内では敵は徐々に強くなっていくイメージ。ラウンド数で一律で敵強度を高めるか、出現する敵ユニット自体を変化させる想定。

### 6.2 エリート部隊
通常の敵勢力とは別に強力な **エリート部隊** がマップ上に出現することがあり、相手からは攻めてこないが撃破すると追加報酬がもらえる。

### 6.3 メタ層：拠点と王国の記憶
拠点は **現代に残る亡国の王国遺跡** そのもの。プレイヤー（主人公）が冒頭で探索した、その遺跡である。

- プレイヤーがラン中に持ち帰った「王国の記憶」を消費することで、当時の姿を取り戻していくイメージ。

#### 6.3.1 メタ通貨「王国の記憶」
メタ進行に使う唯一の通貨として **「王国の記憶」** を定義する。エンディングを見たり強敵を撃破することで多く手に入る。

#### 6.3.2 メタ強化ツリー
拠点には いくつかの施設があり、それぞれに強化ツリーがあるイメージ。各ツリーの強化段階に応じてラン内でのイベントが解禁されていく。

##### 6.3.2.1 訓練所（仮）
- 主な解禁要素：ユニットアンロック、ユニットLv上限強化（直接強化するのではなく、上限を上げる形）

##### 6.3.2.2 図書館（仮）
- 主な解禁要素：装備アンロック、内政施設アンロック、内政コマンドの強化、行動力増加など
- ユニット以外の内政強化を主に担当する

##### 6.3.2.3 神殿
- 固有ユニットの強化：一般ユニットと違って直接強化できる想定。強化には幅があってちょっとしたビルド要素があるとよさそう。
- 固有ユニットの加入に関しては、ラン内で救出すれば一時加入で、その後キャラクターイベントを完走すれば永続化するイメージ

### 6.4 ランリセット原則
ラン開始時、全ユニットのレベルと装備はリセットされる。レベルアップ・装備獲得はラン内で完結し、持ち越しはない。

- メタ進行（拠点強化）によって強化されるのは「**上限**」と「**使用できるユニットの種類**」であり、一般ユニットのラン内の強さそのものではない。
- 固定ユニットは例外。また、一般ユニットからもお気に入りを引き継げるような要素はあってもいいかも。

## 7. 見せ方とプレイ時間

### 7.1 UI/UX 設計指針

#### 7.1.1 セーブ&レジューム
オートセーブはラウンド開始時に自動実行される。これはラウンド単位のやり直し（リロードでそのラウンドを最初からやり直す）の基点ともなる。

手動セーブは以下のタイミングを除いていつでも実行できる：
- 戦闘中
- 会話イベント中

内政フェーズの途中・師団配置の途中・個別戦闘の直前でも保存可能。これにより、2〜3時間級のランを複数セッションに分けて遊べることを保証する。手動セーブはセッションの中断・再開（レジューム）が主目的で、ラウンド単位のやり直しはオートセーブから行う。

#### 7.1.2 失敗時の救済とラン放棄
ゲームオーバー（バッドエンド）時も、そのランで獲得した王国の記憶は **全額持ち帰れる**。途中で力尽きたランが無駄にならないよう設計する。

クリアボーナスが大きいため、最後まで耐えてエンドを迎えることが推奨される遊び方だが、戦況が厳しい場合に **内政フェーズ中に本拠地で「ラン放棄」を選択して次の周回に切り替える** プレイスタイルも許容する：

#### 7.1.3 各フェーズの UX 方針

**師団配置フェーズ**：
- 前ラウンドの配置を引き継ぎ、差分のみ調整すればよい状態をデフォルトとする
- お気に入り編成の保存・呼び出し、ユニット絞り込み検索、まとめて移動

**戦闘観戦**：
- コマ送り・等倍・2倍速・4倍速・スキップを切り替え可能
- 1戦ごとにスキップ選択が可能。「全スキップ」でそのラウンドの残り戦闘をまとめて結果表示

#### 7.1.4 戦闘演出のアート方針（静止スプライト＋エフェクトで魅せる）

戦闘画面のキャラクター演出は **静止スプライト × 視覚エフェクトの重ね合わせ** で構成する。キャラクター個別のアニメーション（攻撃モーション・歩行など）は採用しない。

**設計理由**：
- **オートバトル × 高速・スキップ前提との整合**：プレイヤーは大半の戦闘を 2倍速・4倍速・スキップで流す想定。キャラごとのアニメーション制作は投資対効果が低い
- **アセット差し替え容易性**：多数のユニットを段階的に拡張する設計上、1ユニット = 1イラストの単純構造が最適
- **チビ等身との相性**：チビ 2 頭身であれば静止画でも違和感が少なく、リアル等身が動かない不自然さを回避できる
- **エフェクトで動的演出を担保**：剣閃・矢の軌跡・魔法陣・着弾フラッシュなど、エフェクトを静止スプライト間に overlay することで戦闘の動感を確保する

### 7.2 想定プレイ時間（1ラン2-3時間×3-10周≒30時間）

| 単位 | 目安 |
|---|---|
| 1ラン | 約2〜3時間 |
| ノーマルエンド初到達 | 約3〜4ラン（合計約8〜12時間） |
| トゥルーエンド到達 | 約10ラン（合計約25〜30時間） |

#### 1ランの時間想定
ラウンド数と、各ラウンドに内政・配置・戦闘解決の3フェーズを持つ構造から逆算する。戦闘は倍速・スキップを前提とし、判断に要する時間（内政の取捨選択・師団配置）が主な時間コストとなる。実際の所要時間はプレイスタイルと習熟度に依存する。

### 7.3 失敗の許容と再挑戦の心理設計
本作はローグライトの失敗を「挫折」ではなく **「前進の一歩」** として体験させることを設計上の核とする。

- **王国の記憶は全額持ち帰り**：ゲームオーバーになっても獲得した記憶は失われない。どんなランも無駄にならない
- **バッドエンドの感情設計**：敗北時に王女が責めるのではなく「ありがとう」と送り返す演出により、「大切な人に救われた罪悪感」と「次こそ応えたい」という再挑戦動機を同時に生む

### 7.4 難易度設計（通常／カジュアル／高難易度）

#### 通常モード
デフォルトでは難易度選択を設けない。メタ進行（拠点強化）の蓄積が実質的な難易度調整として機能する設計とする。

#### カジュアルモード（検討）
ストーリーを最後まで体験したいユーザー向けにカジュアルモードを用意する。
- **プロファイル制**：カジュアルモードを選択したプロファイルは通常モードに戻せない
- **調整内容**：全体的な敵の弱体化に加え、王国の記憶の獲得量増加によってメタ強化サイクルを早める

#### 高難易度モード（検討）
トゥルーエンド到達後のやり込み向けに、さらなる高難易度モードの追加を検討する。
