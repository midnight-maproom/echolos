# Echolos VSプロト 戦闘システム仕様

> 本書は VSプロト戦闘システムの仕様。**戦闘解決ロジックの内側**を定義する。
>
> **方針**：(1) シナジーが面白さの主役、(2) 予測可能性、(3) 戦闘ではなくラン構造（戦線の圧）でバランスを取る。
>
> ラン構造・領地マップ・内政・メタ強化等は [310](310_vsprototype_spec.md) を参照。

---

## 0. 設計原則

| 原則 | 内容 |
|---|---|
| ① シナジー主役 | 同属性を集めるほど強い。複数エンジン成立で「引けなかったら負け」を排除 |
| ② 予測可能性 | ルールは画面で完結・プレイヤーが「なぜ負けたか」わかる。ターゲティング選定で迷わせない |
| ③ ラン構造で勝負 | 戦闘は答え合わせ。複数戦線への配分判断が面白さの基盤。戦闘自体に複雑さは要らない |

派生原則：

- **高い床**：シナジーは加算（なくても戦える）
- **緩い天井**：ソフトキャップは寛大に・複数戦線が「最強編成」の枷
- **可読性優先**：見えないルール・過剰なランダム性は排除

## 1. ステータスとダメージ計算式

### 1.1 ステータス（5 項目に集約）

| 種別 | 項目 | 役割 |
|---|---|---|
| スカラー | HP（兵力） | 残存兵力。現在 HP がダメージ・回復スケールに影響 |
| スカラー | ATK | 攻撃力 |
| スカラー | DEF | 防御力（物理/魔法統合・旧 PDEF/MDEF は廃止） |
| スカラー | SPD | 行動順（タイブレーク確定・回避にも一部影響） |
| カテゴリ | 属性 | **シナジー軸（火・水・補助）と地形による DEF 補正にのみ影響**。ダメージ計算の相性・状態異常の属性連動解除はナシ |
| カテゴリ | 攻撃種別 | 近接（近接 vs 近接で反撃が発生・近接 ATK 補正）／遠隔（反撃の授受なし・遠隔 ATK 補正）／補助型は攻撃種別なし |
| カテゴリ | ターゲティング方向 | 前から狙う（最前の敵）／後ろから狙う（最後尾の敵） |

**攻撃 Waza 不所持ユニットの扱い**：タンク等で攻撃 Waza を持たないユニットは、自分の行動順で**防御フォールバック**が自動発動する（§3.5）。

### 1.2 ダメージ計算式

```
ダメージ = (ATK × 倍率 + 定数A) × √HP / (DEF + 定数B + 環境項)
```

設計の意図：

| 要素 | 意図 |
|---|---|
| `√HP` | 「攻撃役の HP 維持に価値」「倒しきれなくても削れば貢献」「終盤の HP インフレで固定回復が陳腐化する問題の解消」 |
| `DEF` が分母 | 減算式廃止。誰でもダメージを出せる「高い床」。防御を盛られて詰む構造を排除 |
| `+ 定数A` / `+ 定数B` | 低ステータスでの式破綻防止＋最低保証 |
| `+ 環境項` | 地形による属性別の DEF 補正（§5） |

**ダメージ式から導かれる帰結**：

- ステータス加算系シナジー（DEF +X 等）に明示的キャップ不要：DEF 無限積みでも分母増加なのでダメージ 0 にならない
- ATK 加算系シナジーも 1 ターン 1 攻撃の自然制約があるためキャップ不要
- 一方、**クリティカル・回避など「式を経由しない非ステータス効果」はキャップ必須**

### 1.3 回復式

```
回復量 = √回復役の現在HP × スキル威力
```

- 対象：「現在 HP / 最大 HP」が最低の味方（直感的・視覚で判別容易）
- 最大 HP 超過分は無効
- **回復役の HP を削る戦術価値**：回復役を狙う敵が出てきたとき、回復役を守る判断が生まれる
- 固定値スケールの終盤崩壊問題を構造解消（HP がどのユニットも育つので回復も自然スケール）

### 1.4 バフ・デバフ式（基礎加算 / 最終出力割合 / 確率パラメタの 3 系統）

バフ・デバフは **作用箇所で 3 系統に明確に分離**する。基礎ステータスへの乗算は禁止し、乗算は最終ダメージ出力にのみ許す。これにより計算序盤の指数的爆発を構造的に防ぎ、係数のコントロール可能性を維持する。

| 系統 | 作用箇所 | 計算 | 例 |
|---|---|---|---|
| 基礎ステータス加算 | ATK / DEF / HP / SPD | 実効ステータス = 基礎値 + Σ(Magnitude × Stacks)（負値はクランプ） | ATK バフ・DEF バフ・ATK デバフ・DEF デバフ等 |
| 最終出力割合 | ダメージ式算出後のダメージ | 加算スタックで一括乗算（§1.4.1 順序参照） | 与ダメ%／被ダメ%／クリ率%／クリティカル倍率 |
| 確率パラメタ | 命中・回避判定 | 加算スタック・キャップあり | 回避%（§1.4.2） |

最終出力割合系（与ダメ%／被ダメ%／クリ率% 等）はパッシブとして守る（永続・解除不可）か、Triggered で意図的に剥がせる設計にするかを Waza ごとに選ぶ。

**ターゲット原則**：直感的・期待通りの挙動（「対象 stat が最も高い味方／敵」が基本）。

#### 1.4.1 最終ダメージ計算順序

```
最終ダメージ = DamageFormula出力
            × (1 + Σ与ダメ%合計)            ← 加算スタック・青天井
            × (1 - min(Σ被ダメ%合計, 0.80))  ← 加算スタック・防御側キャップ 80%
            × クリティカル倍率                ← クリティカル時のみ・最後に乗算
```

設計意図：

- **与ダメ％ → 被ダメ％ の順で乗算**：同数値なら防御側がわずかに有利（例：与ダメ +20% vs 被ダメ -20% で最終 96%）
- **防御側にのみキャップ（暫定 80%）**：「どんなに積んでも 20% は通る」ことで「ダメージが通らない退化戦略」を構造的に防ぐ。攻撃側にキャップなし
- **クリティカルは最後に乗算**：加算スタックの予測可能性を保ちつつ、クリティカルの瞬間的爆発力で「気持ちよさ」を残す折衷案。最終的にクリティカルが重要になるバランス設計
- **乗算は最終出力のみ**：基礎ステータスへの乗算（ATK × 1.5 等）は一切禁止。基礎は加算のみで管理することで計算序盤のレバレッジを抑える

**クリティカル基本値**：

| 項目 | 値 |
|---|---|
| 基本クリ率 | 0%（全ユニット共通） |
| 基本クリ倍率 | ×1.5（全ユニット共通） |
| 付与方式 | パッシブ（永続・解除不可）または一時バフ（Triggered） |

例：焔影の暗殺者はクリ率 +10% をパッシブで持つ。パッシブは Dispel 系で剥がされない。

#### 1.4.2 回避（イベイド）

回避は**確率パラメタ系**（§1.4 表）。命中・回避判定の段階で発動する。

| 項目 | 値 |
|---|---|
| 効果値 | 効果の Magnitude をそのまま % として扱う（Magnitude=10 で 10% 回避） |
| スタック | 加算スタック（複数効果の Magnitude を合算） |
| キャップ | 50%（合算後に上限適用） |
| 対象 | プロト範囲では**遠隔攻撃のみ対象**（攻撃側が遠隔の時のみ回避ロール発火） |

#### 1.4.3 効果付与の暫定既定値

| 効果カテゴリ | 持続 | RemainingTurns | MaxStacks | Cleanse 可否 |
|---|---|---|---|---|
| ATK/DEF デバフ | Triggered | 3 | 1 | 不可 |
| 熱傷（被回復ダウン） | Permanent | -1 | 9 | 可 |
| 炎上（DoT） | Permanent | -1 | 99 | 可 |
| 回避バフ | パッシブ（永続）／一時バフ（Triggered 3T） | -1 / 3 | 1 | 不可 |

「3T / MaxStacks=1」は Triggered 系の**暫定既定値**であり、個別仕様で調整可。付与確率は 100%（命中したら必ず付与）が既定。Cleanse 可なものは Cleanse 系の解除技で剥がれる。

#### 1.4.4 Burn（炎上）の蓄積モデル

| 項目 | 仕様 |
|---|---|
| 発動タイミング | ターン終了時（HoT 回復の前） |
| ダメージ算出 | Stacks × max(1, Magnitude)（固定値・Magnitude は付与 Waza ごとに設定） |
| Shield との関係 | **貫通**（Shield を消費せず HP 直撃。Shield は攻撃ダメージのみ吸収する仕様で、DOT による Shield 削れを防ぐ） |
| 自然治癒 | なし（時限解除なし） |
| スタック上限 | なし（蓄積し続ける） |
| 解除経路 | Cleanse 系（§4.7）のみ |

設計意図：

- 長期戦で価値を維持する DoT に純化
- スタック蓄積で「長く戦うほど無視できなくなる」効果カーブ
- 解除手段は cleanse 系（プロト範囲では癒水の巫女）に限定 → 「炎上対策」が編成判断の軸になる

## 2. 配置ルール

### 2.1 6 体一列（内部スロット）

- 内部スロットは `slot 0`（一番前）から `slot 5`（一番後ろ）の一列
- **配置数 < 6 なら内部計算上は詰めて一列**。空席（飛び石配置）は内部計算には存在しない
- 例：3 体配置時の内部スロットは 0/1/2、4/5 は存在しない扱い

### 2.1.1 配置 UI × 戦闘画面 × 内部計算の 3 レイヤー分離

プレイヤーの自由度と内部計算の単純さを両立するため、以下の 3 レイヤーで責務を分離する：

| レイヤー | 仕様 |
|---|---|
| **配置 UI**（プレイヤー操作） | GUI 上の 6 枠に**好きな順番で配置可能**。空きを作っても良い（強制的に詰めない） |
| **戦闘画面表示** | 戦闘開始時の **GUI 上の並びをそのまま固定表示**。空きはそのまま空きとして表示 |
| **内部計算**（配置 ATK 補正・ターゲティング・反撃判定） | **暗黙的に空きを詰めて計算**。GUI 上の空きは内部スロットには存在しない |

**戦闘中の動的再計算**：
- ユニットが戦闘中に死亡しても、戦闘画面の見た目はグレーアウトのまま固定（位置は動かない）
- 内部計算では死亡ユニットを除外して**詰めて再計算**：残った生存ユニットで内部スロット 0/1/2... を取り直す
- 配置 ATK 補正（特に遠隔の「最後尾からの距離」）・ターゲティング（最前/最後尾敵）が動的に変化する

### 2.2 ターゲティング（核）

**配置 × ターゲティングの 2 軸分離**。攻撃種別（近接/遠隔）は反撃可否＋ ATK 補正方向を決め、ターゲティング方向（前から/後ろから）はターゲット選定方向を独立に決める。ターゲット選定は**内部スロット**（§2.1.1）で行う。組み合わせは 4 通り：

| 攻撃種別 | ターゲティング方向 | 通称 | 戦術的位置付け | 例 |
|---|---|---|---|---|
| 近接 | 前から | 近接前狙い | 普通の前衛アタッカー。反撃を受けつつ最前を削る | サムライ・双剣士 |
| 遠隔 | 後ろから | 遠隔後狙い | 普通の弓兵。反撃なしで敵後衛を狙う | 弓兵 |
| 遠隔 | 前から | 遠隔前狙い | 前列焼き。反撃なし＋遠隔 ATK 補正で「後衛から最前線を削る」 | 炎魔導士 |
| 近接 | 後ろから | 近接後狙い | 暗殺者。前衛で反撃食らうリスクを取って敵後衛をピンポイント | 暗殺者 |

「かばう」「列単位保護」「特定タイプ優先ターゲット」のような不可視ルールは持たない。**プレイヤーは画面を見るだけでターゲットが予測できる**ことを最優先。

### 2.3 配置 ATK 補正（近接）

| slot | 補正 |
|---|---|
| 0 | 100%（タンク席） |
| 1 | 100%（タンク席） |
| 2 | 95% |
| 3 | 85% |
| 4 | 75% |
| 5 | 65% |

設計の意図：

- slot 0-1 を 100% にする＝最前タンクの後ろが近接アタッカーの最適位置になり、反撃を受けず最大火力（§3.4 反撃ルールと合わせて教育的シグナル）
- 中央を境に補正が大きくなる（slot 2→3 で 95→85% にジャンプ）。「3 番目以降は明確にコスト」とプレイヤーが覚えやすい
- 最後尾 slot 5 で 65%（-35%）＝「ほぼ使えない」抑止だが、絶対禁止ではない（緩い制約）

### 2.4 配置 ATK 補正（遠隔）

**「実際に配置された最後尾スロットからの距離」** で計算：

| 最後尾からの距離 | 補正 |
|---|---|
| 0（最後尾） | 100% |
| 1 | 100% |
| 2 | 95% |
| 3 | 85% |
| 4 | 75% |
| 5 | 65% |

設計の意図：

- 6 体編成の slot 4 と 3 体編成の slot 1 は同じ「最後尾から距離 1」で同じ補正
- **少人数編成では補正がほぼ消える**（3 体編成では最大でも -5%）。序盤に弓兵を取るだけで不利になる問題を解消
- 配置 ATK 補正の目的は「近接ばかり並べる／遠隔ばかり並べる退化戦略の抑制」であり、少人数で補正なしは目的と整合

## 3. 行動順と反撃

### 3.1 SPD 順とタイブレーク

- SPD の速い順に行動
- **SPD 同値時のタイブレーク**（完全決定論）：
  1. 陣営間：攻め側陣営が優先（ホームグラウンド優位のフレーバー）
  2. 同一陣営内：slot 番号昇順（最前のユニットから行動）
- **動的順序方式**：行動順はターン開始時に固定せず、**各ステップで「未行動かつ生存」のユニットから実行時 SPD で次の 1 体を選定**する。ターン中の SPD 変動（バフ/デバフ・Freeze スタック増減等）が次の選定に即反映される。ターゲット選定も実行直前に行うため、先行ユニットがターゲットを倒した場合に後続ユニットが別ターゲットを再選定する
- **過剰なランダム性は排除**：SPD 順・行動順・ターゲット選定は完全決定論。クリ率・回避率等の**細かい確率要素は許容**（戦闘結果が予測不能にならない範囲）。ランダム性の主要な提供軸はドラフトと敵編成抽選で、戦闘内のロールは補助的な揺らぎに留める
- SPD は「A で弱体 → B で攻撃」のコンボ成立可否を決める編成設計の軸

### 3.2 反撃

近接攻撃を受けた近接ユニットは即座に反撃する。**攻撃側＆被弾側ともに近接の局面でのみ反撃が発生する**（近接 vs 近接限定）。

| 条件 | 仕様 |
|---|---|
| 反撃発動条件 | 攻撃側が近接**かつ**被弾側が近接（ターゲティング方向は無関係） |
| 遠隔攻撃時 | 反撃なし（弓兵は近接タンクを撃っても反撃を受けない・遠隔の優位性） |
| 被弾側が遠隔の時 | 反撃なし（弓兵が近接タンクに撃たれても反撃しない・タンクは一方的に殴れる） |
| タイミング | 攻撃と同時解決（1 つのアクション内で完結） |
| チェイン | 反撃に対する反撃はない（無限ループ回避） |
| 死亡時 | 反撃で攻撃側が死んだ場合、その時点で打ち止め。攻撃を受けて死亡した側は反撃しない |
| 多段攻撃 | 各 hit ごとに反撃発動（シールド粒度と整合） |
| 範囲攻撃 | 攻撃側が遠隔の範囲攻撃は**反撃なし**（焦熱の大魔導士・暁光の大魔導士等）／攻撃側が近接の範囲攻撃は**被弾側が近接の対象全員から個別反撃** |
| シールド吸収 | 反撃は発動する（攻撃された事実は変わらない） |

ログ表示は**「A の攻撃 → B に xx ダメージ／B の反撃で A に xx ダメージ」を 1 行で表示**することで「同時解決」のメンタルモデルを保つ。

### 3.3 反撃 Waza

- ユニットごとに反撃 Waza を定義可能（特殊効果：強化倍率・毒付与など）
- 未定義時は**共通フォールバック反撃 Waza**（倍率 0.5・属性なし・付帯なし。前列同士の殴り合いが過剰に削れすぎないよう半減で運用）

| 反撃ダメージの計算 | 通常ダメージ式と同じ |
|---|---|
| 配置 ATK 補正 | 乗る |
| 属性バフ | 乗る（染め効果は反撃にも反映） |
| 環境項（攻撃者側の DEF 補正） | 乗る |

### 3.4 反撃が生む戦術的価値

- 旧仕様で ATK=0 だったタンクにも ATK の意味が生まれる（近接 vs 近接の局面で反撃源として機能）
- 近接アタッカーが敵 Melee を攻撃すると反撃を食らう＝最前線は消耗するもの
- slot 1（タンクの後ろ）の近接アタッカーは敵から狙われない＝被弾しない＝反撃側にならず無傷で殴り続けられる最適位置（自分が敵 Melee を攻撃した時は反撃を受ける）

### 3.5 防御フォールバック（攻撃 Waza 不所持ユニットの行動）

攻撃 Waza を持たないユニット（タンク等）は、自分の行動順で**自動的に防御アクション**を発動する。SPD 順は通常通り回る。

| 項目 | 仕様 |
|---|---|
| 発動条件 | 行動評価で**すべての候補が空振り**（攻撃ターゲット・支援ターゲットともになし） |
| 効果 | 自身に DEF +3（暫定）を付与 |
| 持続 | 次の自分の行動順まで |
| ログ・バッジ | 抑制（1:1 対応で自明） |

設計意図：

- タンク（炎の大盾兵・水の大盾兵）は通常攻撃 Waza を持たず防御フォールバックで動くが、**反撃発動者としての近接属性を維持**する（反撃は攻撃側／反撃側ともに近接必須）
- 一方、**攻撃 Waza を一切持たない補助・回復専門ユニット**（炎の鼓舞師／水の護術師／癒水の巫女／光の司祭）は**攻撃種別を持たない**（反撃の授受なし・配置 ATK 補正対象外）
- 反撃は §3.2 の独立した経路で発動するため、防御中でも近接攻撃を受ければ反撃が走る

**ターン中の陣営全滅で即決着**：ターン進行中の各行動後に全滅判定を行い、全滅していれば残りの未行動ユニットは行動させない（無意味な防御発動・攻撃発動を抑止）。さらにそのターンの**ターン終了フェーズ（CD 減算・HoT 回復・Burn 適用・パッシブログ等）も丸ごとスキップ**して即決着する。HoT 回復・Burn ダメージ・熱傷の残り火で戦闘結果が後出しで変わることを防ぎ、「先に倒した側が勝ち」を確定させる。

### 3.6 行動評価方針（推測容易性の原則）

戦闘 AI は §0 設計原則②「予測可能性」に従い、**プレイヤーが事前に「このユニットは何をするか」を予測できる**範囲に留める。複雑な AI 判断ロジックは構造的に持たない。

| 項目 | 仕様 |
|---|---|
| AI スコア評価 | **持たない**（被ダメ予測 vs HP のようなスコア比較は不採用） |
| 支援行動の優先順位判断 | **持たない**（cleanse > heal > debuff > buff のような条件分岐は不採用） |
| 複数 Waza の使い分け | Waza リスト順 + 強制発動フラグ + クールダウンのみで表現 |
| 〇ターンおきの技 | 強制発動フラグ付きの Waza にクールダウンを設定（CD 完了時に確定発動） |

#### 3.6.1 行動決定の優先順位

| 順 | 判定 | 採用条件 |
|---|---|---|
| 1 | 行動不能（麻痺・凍結完全） | 強制待機 |
| 2 | 強制発動 Waza | リスト順で最初に「使用可能（CD=0・回数残り・対象あり）」となった Waza |
| 3 | 通常 Waza | リスト順で最初に使用可能となった Waza |
| 4 | すべて空振り | 防御フォールバック（§3.5） |

設計意図：

- プレイヤーは Waza リスト順を確認するだけで「どの技をいつ撃つか」を予測できる
- 戦闘結果が AI の隠れた判断ロジックで揺れない
- 「交互発動」「HP 閾値回復」等の例外的判断はクールダウン＋強制発動フラグの組み合わせで表現する

### 3.7 Waza の構造

Waza は「行動の枠組み」（SPD・クールダウン・対象指定）と「効果の連鎖」の 2 層で表現する。「攻撃 Waza」「バフ Waza」「回復 Waza」のような分類 enum は持たず、効果の組み合わせで Waza の振る舞いが決まる（攻撃 + 付帯付与・回復 + 解除のような複合も Waza 単位で表現可能）。

#### 3.7.1 対象指定タイプ

| 対象指定 | 説明 |
|---|---|
| 敵単体 | 既定はターゲティング方向に従い 1 体／戦略指定で全敵から選定 |
| 味方単体 | 既定は最低 HP 割合／戦略指定で全味方から選定 |
| 敵全体 | 全生存敵 |
| 味方全体 | 全生存味方 |
| 自身のみ | 自身 |
| 方向 N 体（範囲） | 敵の前 N 体／後 N 体（ターゲティング方向に従う） |

#### 3.7.2 単体対象の選定戦略

単体指定で対象を絞り込むときの戦略。既定以外は全生存対象から戦略に従って 1 体選ぶ（ターゲティング方向は無視）。

| 戦略 | 用途 |
|---|---|
| 既定 | 単体敵＝ターゲティング方向／単体味方＝最低 HP 割合 |
| 最低 HP 割合 | 回復・DEF バフ |
| 最高 ATK | ATK バフ（主力味方）／ATK デバフ（最大脅威の敵） |
| 最高 DEF | DEF デバフ（硬い敵） |

タイブレークは slot 昇順で完全決定論。

#### 3.7.3 攻撃チェーンの順序

攻撃 Waza のチェーンは Effect 同士の依存が強い（命中しなければダメ計算しない・回避時は付帯効果も発動しない・反撃は近接 vs 近接のみ）。順序は以下に固定する：

1. 命中判定（回避% + 必中フラグ・遠隔限定・キャップ 50%）
2. 配置 ATK 補正・地形補正の動的計算
3. ダメージ式適用 → 与ダメ% 乗算
4. 反撃時のみ反撃ダメ% 乗算
5. 被ダメ% 乗算（キャップ 80%）
6. クリ判定（クリ率% / 倍率 ×1.5）
7. シールド消費 → ダメ適用
8. 付帯効果発動（命中時のみ・同 target 限定）
9. 反撃発動（§3.2）

付帯効果（炎上付与・ATK デバフ付与等）は攻撃 Waza の中で「命中時の追撃」として宣言される。シールドが攻撃を完全吸収した場合は付帯効果も発動しない（§4.1 水属性）。

#### 3.7.4 必中フラグ

必中フラグは攻撃 Waza ごとに持つ（Waza 単位の属性）。命中判定のロジックは攻撃チェーン内で完結し、必中フラグが立っていれば回避ロールをスキップする。

## 4. 属性シナジー

**適用範囲：味方陣営のみ**。火・水・光すべてのシナジーは戦闘開始時に味方陣営の生存体数だけをカウントし、味方ユニットにのみ永続バフを付与する。敵側は対象外。

理由：敵編成はランダム抽選（[310 §1.4.1](310_vsprototype_spec.md)）で組まれるため、シナジー発動の偏り（同属性が偶然集まる／分散する）が数値設計を歪めないよう、敵側の強度は ATK/HP/DEF の素の数値だけで調整する。シナジーは「染め編成を組む価値」というプレイヤー側のメカに純化する。

### 4.1 メイン属性（プロト 2 種）

#### 火属性（瞬発・少数精鋭・染める快感）

ATK 上位 N 体に**与ダメ % 増加**バフを付与する。基礎 ATK ではなく最終ダメージ出力に作用する（§1.4.1）。

| 体数 | バフ対象 | 与ダメ % 増加 |
|---|---|---|
| 2 体 | 最 ATK 1 体 | +5%（暫定） |
| 3 体 | 最 ATK 1 体 | +10%（暫定） |
| 4 体 | ATK 上位 2 体 | +20%（暫定） |
| 5 体 | ATK 上位 2 体 | +45%（暫定） |
| 6 体 | ATK 上位 2 体 | +70%（暫定） |

数値はすべて暫定で、バランス調整フェーズで再調整する。

設計意図：

- アタッカー過多を構造抑制（バフ対象は 6 体染めでも 2 体まで）
- 「染め切る価値」を 4→6 のジャンプで明示
- 染めるほど「タンク不在＝速攻押し切り」前傾構成になる
- 基礎 ATK ではなく最終出力に作用（§1.4 の二系統ルール準拠）

#### 水属性（持久・全体・序盤から有効）

味方全員に **DEF 固定値加算バフ**と**シールド**を付与する。DEF は基礎ステータス加算で、シールドは 1 ヒット単位無効化（§1.4 の二系統で前者）。

| 体数 | DEF 加算 | シールド残数 |
|---|---|---|
| 2 体 | +10（暫定） | 0 |
| 3 体 | +15（暫定） | 0 |
| 4 体 | +20（暫定） | 1 |
| 5 体 | +25（暫定） | 2 |
| 6 体 | +30（暫定） | 3 |

設計意図：

- 固定値 DEF バフはスケール悪く序盤向け（持久型のキャラ性）
- シールドは **1 ヒット単位**で無効化。多段攻撃でシールド剥がしが成立（多段がカウンターとして機能）
- 染めるほど「アタッカー不在＝持久戦に持ち込む」構成になる

**シールドの挙動**：戦闘開始時に水属性体数に応じた残数を自陣全味方に付与する（永続・解除不可）。**攻撃ダメージのみシールドで吸収**し、Burn 等の DOT は貫通して HP 直撃する（§1.4.4）。被弾時に残数 -1 で消費し、ダメージを 0 化＋**付帯付与も無効化**（攻撃を完全に防いだ扱い）。ただし反撃は通常通り発動する（攻撃された事実は変わらない）。

#### シナジー段階の補完

3 体・5 体時など発動段階の中間は **明示テーブル方式**（各体数の数値を直接指定）。プロトの火・水暫定値は概ね線形補間に近いが、将来別属性で段階的ジャンプ・対数カーブ等を採用する余地を残すため、補間関数ではなくテーブル参照で実装する。バフ対象数（火属性 1 体／2 体の段階遷移）はテーブルに従う。

### 4.2 補助属性（プロト 1 種・光属性 4 体）

光属性は補助属性の役割で、味方全体の**ターン終了時 HP 割合回復**を発火する。火/水と同じ「自陣カウント発火モデル」だが、効果は控えめで「染めない編成の空き枠の質保持役」を担う。

| 体数 | 効果（ターン終了時・自陣全体・最大 HP × X%） |
|---|---|
| 2 体 | 3%（暫定） |
| 3 体 | 5%（暫定） |
| 4 体 | 8%（暫定） |

| 項目 | 仕様 |
|---|---|
| 発動タイミング | ターン終了時（**Burn 適用後**・先に DOT で死ぬ可能性を確定させてから回復処理） |
| 回復量算出 | round(MaxHP × Magnitude / 100)（暫定 Magnitude=3/5/8） |
| 最大 HP クランプ | 最大 HP 超過分は無効（§1.3 通常回復と整合） |
| カテゴリ | 永続（戦闘開始時に付与） |
| ログ集約 | 陣営単位 1 件に集約（「✚ 味方全体に 光の共鳴 LvN：継続回復 +X/+Y/...」形式）|

設計意図：

- 火（瞬発）／水（持久）に対して、光は「**継続じわじわ回復**」のキャラ性
- 単体ピックしやすい性能（染めない編成の空き枠の質保持役）
- 「補助 1 体の単体性能 ≦ メイン属性 2 体染め時のシナジー効果」を保つことで、メイン染めの価値を維持
- 補助属性は §5.1 で地形補正を受けない（属性軸が「シナジー × 地形」の 2 役割に純化）
- 4 体構成（王女・光の騎士・光の司祭・暁光の大魔導士）のうち王女は固有枠で常時 1 体加入。他 3 体はドラフト経由

### 4.3 ユニット構成（プロト・17 体）

| 区分 | 体数 | 内訳 |
|---|---|---|
| 火属性 | 7 体 | Normal 4（炎の双剣士・炎の弓兵・炎の大盾兵・炎の鼓舞師）／ Rare 2（焦熱の大魔導士・焔影の暗殺者）／ 固有 1（ブリジット） |
| 水属性 | 6 体 | Normal 4（水の剣士・水の弓兵・水の大盾兵・水の護術師）／ Rare 2（水鏡の幻盾兵・癒水の巫女） |
| 光属性（補助） | 4 体 | Normal 2（光の騎士・光の司祭）／ Rare 1（暁光の大魔導士）／ 固有 1（王女） |
| **合計** | **17 体** | Normal 10 / Rare 5 / 固有 2 |

詳細仕様（役割・スキル）は §4.8 に記載。基礎ステータス（HP/ATK/DEF/SPD）と各 Waza の数値は暫定値で別途設定する（バランス調整フェーズで再調整）。

### 4.4 ユニットマトリクス（属性 × 役割）

「同じ役割の中で属性間に強弱がある」表記。火・水・光でクロス：

| 役割 | 火属性 | 水属性 | 光属性（補助） |
|---|---|---|---|
| 前衛アタッカー | ◎（炎の双剣士） | ○（水の剣士・火力控えめ＋ATK デバフ付帯） | ○（光の騎士・サブタンク兼） |
| 後衛アタッカー | ◎（炎の弓兵・炎上付帯） | ○（水の弓兵・火力控えめ＋DEF デバフ付帯） | — |
| レアアタッカー | ◎（焦熱の大魔導士＝前 3 範囲／焔影の暗殺者＝クリ特化） | — | ◎（暁光の大魔導士＝全体魔法） |
| タンク | ○（炎の大盾兵・反撃強化・耐久控えめ） | ◎（水の大盾兵・専守） | ○（光の騎士・サブタンク） |
| レアタンク | — | ◎（水鏡の幻盾兵・敵バフ解除＋遠隔回避） | — |
| 回復 | — | — | ◎（光の司祭・単体中回復） |
| レア回復 | — | ◎（癒水の巫女・全体回復＋デバフ解除） | — |
| バフ役 | ◎（炎の鼓舞師・単体 ATK+） | ◎（水の護術師・単体 DEF+） | — |
| 固有 | ブリジット（前衛・固有スキル未定） | — | 王女（前衛・固有スキル未定） |

各ユニットは **単独属性**。「複数属性持ち」はレア限定の特殊枠として別途検討余地あり。

### 4.5 染め編成と R 別配置体数の整合

[310 §1.4.1](310_vsprototype_spec.md) の R 別配置体数（毎 R +2）と発動段階が自然に整合：

| R | 1 戦線あたり配置 | 発動可能なシナジー段階 |
|---|---|---|
| R1 | 2-3 体 | 2 体段階 |
| R2 | 3-4 体 | 2 体段階 |
| R3 | 4 体 | 4 体段階に到達 |
| R4 | 4-5 体 | 4 体段階 |
| R5 | 5-6 体 | 4-6 体段階 |
| R6 | 6 体 | 6 体染め可能（運次第） |

「6 体染めはロマン、現実は 4-2 でやりくり」のバランス目標と整合。

### 4.6 本番想定

- メイン属性 5-6 種、補助属性 3-4 種、各属性 6 体構成で 50 体規模
- 属性数はラン中の手駒数（手駒数 ÷ 6 で 1 属性染めの期待値）から逆算する

### 4.7 効果型階層と解除経路

効果は**振る舞い軸**と**持続/解除の交差軸**の組み合わせで表現する。「能力デバフ」「状態異常」のような総称分類は持たず、振る舞いカテゴリ × 独立フラグで一意に決まる。

#### 4.7.1 効果カテゴリ（振る舞い軸）

| カテゴリ | 例 | 役割 |
|---|---|---|
| 能力値モディファイア | ATK バフ / ATK デバフ / DEF バフ / DEF デバフ | 実効ステ計算で `基礎値 + Σ(Magnitude × Stacks)` を加算 |
| 回避モディファイア | 回避% アップ / ダウン | 命中・回避判定で % 値として加算（合算後 50% キャップ） |
| ダメージ計算モディファイア | 与ダメ% / 被ダメ% / クリ率% / 反撃ダメ% | 最終ダメージに乗算（被ダメは防御側キャップ 80%） |
| 受け回復モディファイア | 熱傷（1 Stack -10%・最大 9） | 最終回復量に乗算される割引 |
| 継続効果（DoT/HoT） | 炎上（DoT・シールド貫通・蓄積）／継続回復（HoT・ターン終了時 HP%） | ターン終了時に HP 操作 |
| 行動阻害 | 凍結（SPD 低下＋ Stacks 閾値で行動不能）／麻痺（許容量倍化）／呪い（HP 閾値で即死） | 行動順時の判定で発動 |
| メーター | シールド（残数・1 ヒット消費・DOT 貫通） | 被弾時に消費 |
| フラグ | 反撃禁止（被弾側基準）／反撃無効（攻撃側基準）／復活無効 | 条件発動の鍵 |
| 自己防御 | 防御中バフ（次の自分の行動順まで） | フォールバック専用 |

新カテゴリ追加（ステ入替・Waza 封印・予約攻撃等）は既存処理（ダメージ式 / Dispel / Cleanse 等）に影響しない方針（Open-Closed）。

#### 4.7.2 持続・解除の交差軸

カテゴリと直交する 4 軸：

| 軸 | 用途 |
|---|---|
| 持続 | 一時（残ターンカウントダウン）／永続（自然消滅なし） |
| Dispel 不可フラグ | パッシブ・オーラに付ける（敵の Dispel 系で剥がされない） |
| Cleanse 可フラグ | 「状態異常」相当に付ける（Cleanse 系で剥がせる） |
| オーラ起源 | 発生源（特定ユニット）が戦闘不能になると剥奪される（連携・固有オーラ） |

#### 4.7.3 解除経路

Waza 側の効果種類で対象が決まる 3 系統：

| 解除系 | 対象 |
|---|---|
| Dispel デバフ系 | 味方のモディファイア系デバフ（Dispel 不可フラグ無し） |
| Dispel バフ系 | 敵のモディファイア系バフ（Dispel 不可フラグ無し）。パッシブオーラは剥がせない |
| Cleanse 系 | Cleanse 可フラグ付き（炎上・凍結・麻痺・呪い・熱傷）を一括解除 |

設計意図：

- 解除経路を 3 系統に分けることで「何が剥がれるか」をプレイヤーに予測可能にする
- パッシブオーラは Dispel から守られ、「状態異常」相当は別経路（Cleanse）に隔離。これにより「炎上対策の Waza ピック」が編成判断軸として機能する

#### 4.7.4 スタックとマージ

同カテゴリの効果が複数付与される場合の挙動は**付与源（スキル単位）で分離**：

| 状況 | 挙動 |
|---|---|
| 同カテゴリ ＋ 同付与源 | 既存効果に Stacks 加算（MaxStacks クランプ）＋ Magnitude / 残ターン数を最新値で上書き＝**リフレッシュ**（鼓舞の再付与・炎上の蓄積等） |
| 同カテゴリ ＋ 異なる付与源 | 別効果として**共存**（連携「ATK +3 / 永続」と鼓舞「ATK +6 / 残 3T」が両方加算される） |

設計意図：オーラ系（永続）と Waza バフ（一時）が同カテゴリでも独立して効くようにする。バッジ UI はホバーで個別の Stacks / 残ターン数 / 付与源を表示。

#### 4.7.5 付与拒否（個別カテゴリ単位の無効化）

ユニットごとに「無効化する効果カテゴリの集合」を持つ。「全状態異常無効」のような総称概念ではなく、個別カテゴリの集合で表現することで「何を無効化するか」が一意に決まる。

| 表現 | 集合内容 |
|---|---|
| 燃焼無効 | { 炎上 } |
| 凍結無効 | { 凍結 } |
| 全状態異常無効 | { 炎上, 凍結, 麻痺, 呪い, 熱傷 } |

#### 4.7.6 プロト範囲の解除技

| Waza | 解除系 | 解除対象（実プレイ挙動） |

### 4.9 オーラ（固有ユニットの陣営常時バフ）

固有ユニットが陣営に在席している間、戦闘開始時に陣営全員へ永続効果を 1 回付与する機構。属性シナジー（§4.1-§4.2）と同じく戦闘開始時に一度だけ評価し、戦闘中の動的再評価は行わない。

#### 4.9.1 対象（プロト範囲）

| 固有ユニット | オーラ名 | 基本効果 | 発動条件 | 配布範囲 | メタ強化連動 |
|---|---|---|---|---|---|
| 王女 | 王家の加護 | DEF +3 | 王女在席 | 陣営全員 | メタ強化「王家の加護 +2」を 1 段階取るごとに +2（最大 2 段階で +4） |
| ブリジット | 連携 | ATK +3 | ブリジット＋王女両方在席 | ブリジット＋王女のみ | なし（固定 +3） |

#### 4.9.2 メタ強化との連動

王女のメタ拠点 Lv 強化（3 択モーダル）から「王家の加護 +2」を選ぶと、王女の強化リストに積み上がる。戦闘開始時に王女の強化リストから「王家の加護 +2」の段階数を読み取り、オーラの基本効果に加算した値を陣営全員に付与する。

- 強化 0 段：味方全体 DEF +3
- 強化 1 段：味方全体 DEF +5
- 強化 2 段：味方全体 DEF +7

王女メタ強化は [30, 60] の 2 段階＝両段階で「王家の加護 +2」を選択すれば最大強化。攻めも守りも選べる 3 択（ATK +5 / HP +20 / 王家の加護 +2）の中で取捨選択する設計。

#### 4.9.3 仕様

| 項目 | 仕様 |
|---|---|
| 対象範囲 | 味方陣営のみ（敵陣営にオーラ機構は適用しない・シナジー方針と同じ偏り排除） |
| 発火タイミング | 戦闘開始時（属性シナジー付与の直後） |
| 付与効果のフラグ | 永続 ＋ Dispel 不可（敵の Dispel バフ系で剥がせない） |
| 発生源戦闘不能時 | オーラ発動なし（戦闘開始時に既に倒れているケース） |
| パートナー必須指定 | 連携系で使用。1 体でも欠ければ不発 |
| 配布範囲 | 陣営全員（王家の加護）／発生源＋パートナーのみ（連携） |
| メタ強化連動 | あり（王家の加護）／なし（連携） |
| オーラ名 | 戦闘開始集約ログ・効果バッジに付与源として表示 |
| 動的解除 | 発生源または必須パートナーが戦闘不能になった時点で、陣営から該当オーラ起源の効果を削除する。王女死亡で味方全体の DEF +N が剥がれ、同時に連携も剥がれる |

## 5. 地形（環境項）

### 5.1 地形の種類（プロト 3 種）

| 地形 | 効果 |
|---|---|
| 水有利 | 水属性ユニットの DEF +α、火属性の DEF -α |
| 火有利 | 火属性ユニットの DEF +α、水属性の DEF -α |
| 中立 | 補正なし |

α の値は層別強度（§5.2）で変化する：

| 強度 | α 値 |
|---|---|
| Light（自領） | 5（暫定） |
| Medium（敵領） | 10（暫定） |
| Heavy（敵拠点） | 15（暫定） |

数値は暫定で、バランス調整フェーズで再調整する。補助属性（光等）は地形補正を受けない（属性軸が「シナジー × 地形」の 2 役割に純化されているため・§1.1）。

**重要原則**：

- **属性同士の相性（水 → 火 の直接相性）はナシ**。あるのは地形バイアスのみ
- これによりカウンター構造が裏口から戻ることを防ぐ

### 5.2 地形の配置ルール（マップ全体）

- **列ごとに地形種類を決定**（3 列 × 各 1 地形）
- 同列内（自領・敵領・敵拠点）は同じ地形種類
- **層によって強度変化**：自領 = 軽微 / 敵領 = 中 / 敵拠点 = 重
- **ランごとに列地形ランダム**（推奨：必ず全 3 属性 1 列ずつを保証する制約パターン、運ゲー回避）
- 本拠地は **中立固定**

### 5.3 環境項はダメージ式の分母にのみ作用

```
ダメージ = (ATK × 倍率 + 定数A) × √HP / (DEF + 定数B + 環境項)
                                                      ↑
                                                  環境項はここ（防御側）
```

帰結：

- 攻撃力には影響しない＝アタッカーの属性は地形と無関係に機能
- **敵側はタンクだけ地形属性追従で編成として機能**（敵編成設計の単純化）
- **プレイヤー側はタンクだけ地形追従＝属性シナジー分断**のジレンマが発生
  - もう 1 体タンクと同属性を入れて 2 体染めを発動するか、シナジー諦めて地形優先するか
- アタッカーも完全地形無視はできない緩やかなインセンティブ（反撃で削られると √HP 価値が下がる）

**分母 1 クランプ**：負の環境項と DEF=0 の組み合わせで分母が 0 以下になる事故を防ぐため、`DEF + 定数B + 環境項 < 1` のときは分母を 1 にクランプする。被ダメ青天井を許容しつつ計算事故だけを構造的に防ぐ（例：水属性・火有利 Heavy・DEF=0 で分母 0 + 10 + (-15) = -5 → 1 にクランプ）。

### 5.4 敵編成の地形追従（中程度）

- 火有利地形 → 火属性タンク含む編成（火属性 4 体程度 + 中立 2 体程度）
- 水有利地形 → 水属性タンク含む編成
- 中立地形 → 属性不問
- **6R 中ボスも地形追従**：地形にマッチした属性で出現＝プレイヤーは「中ボス対策の染め」を計画する

### 5.5 戦線分配との関係（戦略レイヤー）

ラン開始時の意思決定が生まれる：

1. 列地形を確認
2. 染めたい属性に合う列を「攻め列」に決定（敵拠点まで攻める）
3. 他の列は「防衛のみ」で薄く配置
4. → 染め編成は 1 戦線集中、他戦線は混合 4-2 で持たせる分配が最適解になる

---

## 6. 引き分け評価（攻め守り非対称）

タイムアウト時の評価ルール：

| 状況 | 評価 |
|---|---|
| 攻め側で 1 体も敵を倒せず時間切れ | **負け** |
| 守り側で 1 体も味方を倒されず時間切れ | **勝ち** |

攻め／守りの判別は §3.1 SPD タイブレークと共通の概念で行う。マップノード種別からラン進行側が判定して戦闘側へセットする。

設計意図：

- 水 6 染め持久型の「タイムアウト引き分け狙い」退化戦略を構造で塞ぐ
- 攻め側は「最低 1 体は倒す」プレッシャー＝持久戦に逃げられない
- 守り側は「耐え切れば勝ち」＝防衛の価値が明確

---

## 7. 退化戦略防止策の一覧

設計段階で潰した退化戦略：

| 退化戦略 | 防止策 |
|---|---|
| 染め 1 戦線集中＋他放置 | 自領陥落時の本拠地戦（[310 §1.5](310_vsprototype_spec.md)・連戦廃止＋ R 帯切替）／地形ランダムで「攻め列」が毎ラン変わる |
| 近接アタッカー後ろ並べ | 配置 ATK 補正（slot 5 で 65%） |
| 遠隔アタッカー前並べ | 同上（最後尾からの距離） |
| サンドイッチ強要 | タンクの絶対数が少ない＋他の三すくみ構成と相対化 |
| 水 6 染め持久型 | 引き分け非対称評価（攻め側 0 撃破 = 負け） |
| 火 6 染め速攻型 | アタッカー 2 体バフ制限＋染め切れない場合の脆さ |
| 染め編成が常勝 | 敵側も地形に追従して染め編成化、対等の勝負 |
| 全 3 列同属性出現 | 地形ランダムに「全 3 属性 1 列ずつ」制約 |
