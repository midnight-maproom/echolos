# 第1部：検証ツール仕様

Editor メニュー「Echolos/Tools/Balance Report (...)」から実行する Editor 拡張群。
共通基底は [BalanceReportTool.cs](../Assets/Scripts/Presentation/DevTools/Editor/BalanceReportTool.cs)（`MaxTurns` 等の共通定数）。

## 1.1 ツール一覧

| ツール | 用途 | 編成軸 | 集計対象 |
|---|---|---|---|
| [WeakPoolBalanceReportTool](../Assets/Scripts/Presentation/DevTools/Editor/WeakPoolBalanceReportTool.cs) | 弱プール検証 | 偵察兵 2 + ランダム差分 1 × 味方 2 体ペア（自動加入 2/R 化以前の旧体数想定・要再設計） | 味方／敵ユニット別勝率（Front2／Std 合算） |
| [MidPoolBalanceReportTool](../Assets/Scripts/Presentation/DevTools/Editor/MidPoolBalanceReportTool.cs) | 中プール検証 | フル分業 3v3（タンク + 前衛 + 後衛・自動加入 2/R 化以前の旧体数想定・要再設計） | 味方ユニット別勝率 |
| [SixVsSixBalanceReportTool](../Assets/Scripts/Presentation/DevTools/Editor/SixVsSixBalanceReportTool.cs) | 強プール想定 6v6 | 固定枠（重装＋双剣＋入れ替え／炎魔＋入れ替え×2）× 敵 12 編成 = 2592 戦闘 | 入れ替え枠ユニット別勝率 |
| [SixVsSixBackTankBalanceReportTool](../Assets/Scripts/Presentation/DevTools/Editor/SixVsSixBackTankBalanceReportTool.cs) | 後列タンク構成での 6v6 | 後列固定を重装兵に差し替え／後衛入れ替え 2 枠化（味方 294 × 敵 75 = 22050 戦闘） | 入れ替え枠ユニット別＋特定マッチアップ別 |

> 詳細仕様はソースのファイル先頭コメントを SSoT とする。

## 1.2 共通仕様

- **乱数**：`random0to99: () => 50` で固定（命中・回避・クリティカルを中央値固定で再現性確保）
- **最大ターン**：`BalanceReportTool.MaxTurns`
- **勝敗判定**：`BattleResult.PerfectVictory` or `AdvantageousVictory` を味方勝利扱い

# 第2部：検討中・仮置きの仕様

確定設計フレームは [310 §1.4.1](310_vsprototype_spec.md) を SSoT とする。本節は仮置きの具体構成のみ。

## 2.1 敵編成プール内訳（仮置き・最終決定はバランス調整時）

### 2.1.1 弱プール（R1-2 帯・3 体編成）

「偵察兵 2 体 + ランダム差分 1 体」を基本枠とし、差分位置（前列／後列）と差分ユニットの 2 軸で 7 通りを生成する。狙いは「味方 2-3 体で勝てる」基準ライン。

| パターン | slot 0 | slot 1 | slot 2 / 4 | 候補数 |
|---|---|---|---|---|
| P1（前列差分） | 帝国偵察兵 | ランダム前衛差分 | 帝国偵察兵（slot 2） | 4 |
| P2（後列差分） | 帝国偵察兵 | 帝国偵察兵 | ランダム後列差分（slot 4） | 3 |

| 軸 | 候補 |
|---|---|
| ランダム前列差分 | 帝国重装兵・帝国騎士・帝国双剣士・帝国傭兵 |
| ランダム後列差分 | 帝国炎魔導士・帝国弓兵・帝国司祭 |

**ランダム候補からの除外ユニット**：

| ユニット | 除外理由 |
|---|---|
| 帝国偵察兵 | 弱プールの基本枠として固定済（重複させない） |
| 帝国暗殺者 | 後列直撃（かばう貫通・1 回限）が弱プールには不釣り合いな決め技 |
| 帝国大魔導士 | チャージ式全体魔法（強プール想定の脅威） |
| 帝国の影 | 麻痺＋回避で序盤ペアでは対処困難・命中重視ユニット強制圧が偏る |

### 2.1.2 中プール（R3-4 帯・4 体編成）

「タンク P + 前衛 P + 後衛 P + 偵察兵」のコア構造で 4 体編成を組み、各プールから抽選する 18 編成を生成する。R3-4 では味方が 1 マス 4-5 体配置前提のため、対 4 体で検証する。

| プール | 候補 |
|---|---|
| タンク P | 帝国重装兵・帝国騎士 |
| 前衛 P | 帝国双剣士・帝国傭兵・帝国暗殺者 |
| 後衛 P | 帝国炎魔導士・帝国弓兵・帝国司祭 |

| 体数 | 構成 | スロット配置 | 組合せ数 |
|---|---|---|---|
| 4 | タンクP + 前衛P + 偵察兵 + 後衛P | 0=タンクP / 1=前衛P / 2=偵察兵 / 3=後衛P | 2×3×3 = **18** |

中プールで初登場するユニット：**帝国暗殺者**（前衛 P）。後列直撃（疾風刃のかばう＋列単位保護貫通）の脅威が中プールから現れる。帝国の影・帝国大魔導士は強プール残置。

### 2.1.3 強プール（R5-6 帯・6 体編成）

R5-6 では味方が 1 マス 5-6 体配置可能になるため、敵側もフル編成 6 体で対抗する。中プールに対し以下を追加：

- **帝国の影**（中射程・麻痺＋回避 15%）が初登場：状態異常治癒の価値が問われる
- **帝国大魔導士**（全体魔法サンダー mult 0.8・CD3）が初登場：複数体への AoE 圧

| プール | 候補（仮置き） |
|---|---|
| タンク P | 帝国重装兵・帝国騎士 |
| 前衛 P | 帝国双剣士・帝国傭兵・帝国暗殺者 |
| 後衛 P | 帝国炎魔導士・帝国弓兵・帝国司祭・帝国大魔導士 |
| 妨害枠 | 帝国の影（中射程・麻痺・回避） |

**構成は仮置き・詳細決定はバランス調整フェーズ**。コンセプト案：

- E-A 重装型：タンク × 2 + 前衛 × 2 + 後衛 + 偵察兵（壁厚で長期戦）
- E-B 全体魔法型：タンク + 前衛 + 大魔導士 × 2 + 後衛 + 偵察兵（AoE 連発）
- E-C 妨害精鋭型：タンク + 暗殺者 + 影 × 2 + 後衛 + 偵察兵（麻痺＋暗殺で陣形崩壊）

> 中ボス（男爵・隻眼）の取り巻きは強プールの延長として扱わず、別カテゴリで [BossesRoster](../Assets/Scripts/Domain/Prototype/BossesRoster.cs) 既存実装をそのまま使う。

## 2.2 場所補正（ハイブリッド：係数 + 一部差し替え）

### 2.2.1 ステータス係数

| 場所 | HP 係数 | ATK 係数 | コメント |
|---|---|---|---|
| 自領 | 1.00 | 1.00 | プレイヤーが防衛する側＝攻めてくる敵の基準値 |
| 敵領 | 1.10 | 1.05 | 領地を守る側がやや上 |
| 敵砦 | 1.20 | 1.10 | 拠点防衛は最強 |

### 2.2.2 構成差し替え

| 場所 | 差し替え | 狙い |
|---|---|---|
| 自領 | なし | プールそのまま |
| 敵領 | なし | プールそのまま（係数のみで差別化） |
| 敵砦 | 編成に **帝国重装兵 1 体追加 or 1 体精鋭化**（帝国偵察兵 → 帝国双剣士など） | 拠点は壁役と精鋭ユニットで固められている演出。プレイヤーの編成判断を強制 |

> 「精鋭化置換ルール」はバランスツール検証で詰める。最終的に「同じ抽選編成でも場所により 1 体だけ差し替わる」前提だが、置換マッピングの妥当性は要検証。

## 2.3 ラウンド係数（仮値・要検証）

旧戦闘プロト経験：β案（通しで R ごと ×1.05）採用時、敵強度上昇に味方強化が追いつかず難易度カーブが破綻した経緯あり。VSプロトでは強化仕様・行動力仕様・メタ強化が変わっているため、再検証が必要。

### 2.3.1 仮値（β案踏襲）

| R | 係数 |
|---|---|
| R1 | 1.00 |
| R2 | 1.05 |
| R3 | 1.10 |
| R4 | 1.16 |
| R5 | 1.22 |
| R6 | 1.28 |

### 2.3.2 検証方針

検証ツール群を使い、各 R のプレイヤー想定編成（ドラフト + 強化 + メタ強化込み）vs 同 R 敵編成（プール × 場所 × R 係数）で勝率を測定。R が進むごとに「拮抗 ±10%」程度を維持できるよう係数を調整。

R6 で味方強化が追いつかない兆候が出たら係数を緩める or 上昇カーブを線形 → 対数化する。

## 2026-06-13 セッション

### 個別ユニット調整

| ユニット | 変更 | 理由 |
|---|---|---|
| 大槌兵 | 鎧砕き mult 0.8→1.0／DefenseDown Magnitude 5→7 | 6v6 で 51.9% → 64.4%（前衛平均 +13）。0.8 倍火力 + DefDown のリターンが見合っておらず、火力底上げ＋デバフ強化で「タンクキラー」の本領を発揮 |
| 巫女 | 祈り amount 7→12 | 6v6 で 55.2% → 67.6%。司祭の単体 45 と棲み分けるピンチ集団回復役へ |
| 軍師 | 戦術隊形 Magnitude 3→5 | 6v6 で 51.4% → 61.7%。BackTank では再度 35.7% に下落。状況依存技 2 個＋オーラ構成が「敵バフ・味方デバフ大量発生」前提だが、プロト環境では発生しない |
| 司祭 | 治癒の光 amount 45→35 | BackTank で 69.9% → 65.3%。長期戦での回復価値が二乗的に高まり Unbalanced 寄り。総量縮減で他後衛と相対化 |
| 帝国司祭 | amount 45→35 | 味方司祭との同期 |
| 忍者 | HP 95→105（+10） | BackTank で 26.7% → 40.8%。115（+20）試した後、過剰の可能性で半減 |
| 帝国暗殺者 | HP 95→105 | 味方忍者との同期 |

| ツール | 内容 |

### 検証結果スナップショット

**6v6 BackTank（最終版・全体勝率 49.4%）**

味方前衛：サムライ 81.6% / 王女 47.1% / 大槌兵 45.3% / 双剣士 42.0% / 忍者 40.8% / 騎士 39.4%

味方後衛：司祭 65.3% / 踊り子 53.3% / 炎魔導士 50.2% / 弓兵 49.3% / 巫女 46.9% / 雷魔導士 45.6% / 軍師 35.7%

敵：帝国傭兵 79.4% / 帝国司祭 58.7% / 帝国炎魔導士 57.3% / 帝国の影 52.9% / 帝国大魔導士 43.9% / 帝国弓兵 40.4% / 帝国騎士 36.4% / 帝国暗殺者 36.0%

特定マッチアップ：
- 忍者 vs (帝国炎魔導士 or 帝国大魔導士)：2352 戦闘・**55.2%**（全体平均 +5.8pt・魔導士キラー役割が機能）
- 巫女 vs 帝国傭兵：1950 戦闘・**25.9%**（帝国傭兵自体が強すぎて全体回復のカウンターでも押し切られる）

### 観察と保留事項

- **サムライ／帝国傭兵が突出（+30pt）**：BackTank では前衛固定が双剣士＝柔らかい敵が確定的に居る構造で過評価の可能性。通常 6v6 では +1.8pt 程度。構造起因と純粋な強さの切り分けが要る → 前衛固定枠バリアントの追加 or スプラッシュ係数 0.8→0.7 試行が候補
- **軍師の構造問題**：状況依存技 2 個＋オーラの構成自体がプロト環境で機能不全。ボス側を「バフ・デバフ多めの設計」にすることで「対ボス特効」役として救う方針で保留
- **巫女 vs 帝国傭兵 25.9%**：全体回復は機能しているが帝国傭兵自体が強すぎる

### 次セッションへの引き継ぎ（バランス関連）

- 中プール検証ツールの 4v4 再設計（自動加入 2/R 化に対応）
- 弱プール検証ツールの 2-3 体対応（同上）
- 強プール詳細決定（E-A/B/C コンセプト案の絞り込み）
- サムライのスプラッシュ係数下方／前衛固定枠バリアントによる構造影響切り分け
- ボスデザイン時にバフ・デバフを増やして軍師の役割を確立
- 場所補正係数・R 係数の数値検証
- ランダム抽選ロジック実装（`VSPrototypeEnemyPatterns` 差し替え）
