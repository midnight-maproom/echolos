## 5. 実装ステップ（推奨順）

### 5.1 Step 4-1：ユニット用アイコン素材の選定とインポート（ユーザー作業）
- 通常15体＋ボス2体＋散兵＝18枚程度のアイコン素材
- フリー素材：itch.io / OpenGameArt / Kenney などから世界観に近いものを10〜20分で決める
- プロト用と割り切り「無難に選ぶ」を最優先（沼に入らない）
- **配置先**：`Assets/Resources/Icons/{unitId}.png`（Resources フォルダから実行時ロード）
- **ファイル名**：`unitId.png` 固定。例：`s3_tank_def.png`（重装兵）、`s3_atk_multi.png`（双剣士）
- **推奨サイズ**：64×64〜128×128 ピクセル。PNG / JPG どちらも可（描画側でアスペクト比保持して内接スケール）
- スキャフォールド（`Stage3IconRegistry`）が遅延ロードしキャッシュ。配置漏れの兵種はプレースホルダー矩形＋名前テキストに自動フォールバック

### 5.2 Step 4-2：戦闘可視化（最重要・Claude 主担当）
- **観戦専用フェーズ**（`Phase.Spectating`）でフルスクリーン表示。配置直後の「戦闘実行 →」で観戦に遷移し、戦線ごとに順次再生する
- **レイアウト**：縦3行×横2列でレーン向き合い。味方は左=後列・右=前列、敵は左=前列・右=後列。中央軸で前列同士がレーン（0,3）（1,4）（2,5）で対峙する JRPG 風配置
- **各スロット**：アイコン or プレースホルダー矩形＋ユニット名＋HPバー（緑→黄→赤グラデーション＋数値）
- **行動者ハイライト**：直近の行動者を黄色でハイライト
- **被弾フラッシュ**：被弾対象を赤フラッシュ＋ダメージ値（「-12」「☠5」「回避」）を右上に表示
- **死亡時**：スロットをグレーアウト＋名前に「×」前置
- **結果オーバーレイ**：戦闘終了時に結果チップ（完勝＝緑／辛勝＝橙／惜敗・完敗＝赤）
- **構造化イベント記録**：`Stage3BattleRunner` が `Stage3BattleEvent` 列を記録し、観戦ビューが時間軸で再生する録画方式（後段の倍速・ステップ制御の基盤）
- ターゲット選定の結果（誰を狙ったか）が視覚で見えること＝USP②（配置が結果を決めるオートバトル）の「読み」の核心

### 5.3 Step 4-3：テンポ調整（Claude 主担当）
- **速度トグル**：ステップ／×1／×2／×4（基準 0.5s/イベント を倍率で除算）
- **ステップモード**（`_spectateSpeed == 0`）：自動 Tick を停止し、「次イベント →」ボタンでユーザーが1イベントずつ手動送り。じっくり戦況を観察したいとき・スクショ撮影時の止め絵用
- **この戦闘をスキップ**：現在の戦闘の残りイベントを一気に消化して最終状態へ
- **すべてスキップ**：全戦線の観戦を打ち切って戦闘結果フェーズへ
- **自動進行**：戦闘終了から 1.2 秒後に次戦線へ自動遷移（ステップモード時は無効化）
- 速度・自動進行設定は複数戦闘をまたいで保持される
- 動画撮影は取り下げたが、企画書スクショ撮影の止め絵調整に効く

### 5.4 Step 4-4：OnGUI スタイル磨き（Claude 主担当）
- **中央集約テーマ** `Stage3GuiTheme`：色パレットと GUIStyle を一箇所で管理。ビュー間のスタイル散らかりを防ぐ
  - 色：`PanelBg`（濃い青みグレー）／`AccentPrimary`（青）／`AccentSuccess`（緑）／`AccentWarning`（橙）／`AccentDanger`（赤）／`TextPrimary`（オフホワイト）／`TextMuted`（薄グレー）
  - スタイル：`Title` 22pt 太字／`Subtitle` 16pt 太字／`Body` 13pt（wordWrap 有）／`Muted` 12pt 薄色／`Stat` 15pt 太字（数値強調）／`Panel` 統一背景／`PrimaryButton` 16pt 太字
  - ヘルパー：`DrawChip(text, color)` 色分けタグ／`PhaseColor(label)` フェーズ別アクセント／`FillRect` 塗り背景
- **適用箇所**：
  - キャンペーンヘッダ（ラウンド大見出し・行動力 Stat 強調・フェーズ色分けチップ）
  - 主アクションボタン（戦闘実行 → ／ 次のラウンドへ → ／ 内政終了 ／ 新規スタート）を PrimaryButton 化
  - 観戦ビューヘッダ（戦線名 Subtitle・ターン Stat・結果チップ色分け）
  - 観戦ビュースロット文字（毎フレーム new GUIStyle を発行していた処理をキャッシュ化）
- 配色は寒色系をベースに、アクセント色だけで状態を伝える方針

### 5.5 Step 4-5：チュートリアル／ガイド（Claude 主担当）
- **フェーズヘルプ枠（A案）**：手駒パネル直下に「? このフェーズで何をする」チップ＋操作説明を常時表示。右上「▲ 隠す / ▼ 展開」で折り畳み可（動画やスクショ撮影で邪魔なら畳める）。テキストは `Stage3GuideContent.GetPhaseHelp(phaseLabel)` から取得
- **ドラフト詳細カード**：3択ドラフトの各候補を 230×300 の詳細カードで表示。役割チップ／射程／属性／ステータス1行／▼役割サマリ／▼技一覧（`WazaOneLineSummary` 整形）／▼おすすめ／「この兵を選ぶ」主ボタン。初期ドラフト・ラウンドドラフト・招集すべての3択で同じカードを使用
- **手駒パネルの仕様書順ソート**：`Stage3GuideContent.UnitSpecOrder`（前衛7→後衛8→ボス→雑魚）で安定ソート。配置フェーズで「あの兵どこ？」の探索コストを下げる
- 別画面の常設ヘルプ画面は実装しない（フェーズ枠で代替・コスト削減方針）
- 初回ラン限定の切替も実装しない（折り畳み可で常時表示・繰り返しプレイ時の邪魔は畳むことで吸収）

### 5.6 Step 4-6：WebGL ビルド＋itch.io 限定公開（ユーザー作業・2026-06-02 方針変更）
**変更理由**：当初は Windows EXE で配布予定だったが、知人や審査員への試遊体験を考えると
ブラウザ実行（WebGL）の方が摩擦が少ない（zip送付・展開・未署名exe警告が全て不要・
Mac/Linux ユーザーも触れる）。性能も本作の軽量さなら十分。

**Unity 側操作**：
1. **Unity Hub** → Installs → 該当バージョンの歯車 → Add modules → **WebGL Build Support** にチェックしてインストール（数百MB・初回のみ）
2. **Noto Sans JP TTF** を Google Fonts から取得し、`Assets/Resources/Fonts/NotoSansJP-Regular.ttf` として配置(WebGL でも日本語が表示されるよう `Stage3GuiTheme.EnsureJapaneseFont()` が自動で全 GUI に適用する）
3. **File → Build Settings** → Platform = WebGL → Switch Platform（数分）
4. **Player Settings**：
   - Publishing Settings → Compression Format = Gzip（itch.io 推奨）／Decompression Fallback ON
   - Resolution and Presentation → Canvas **1920×1080**（2026-06-04 拡大・GPT イラスト縮小品質を緩和するため）
5. **Scenes In Build** は `EcholosProto_Main.unity` のみ
6. **Build And Run**（または Build → 出力フォルダ `Build/WebGL/` を指定）。初回は IL2CPP 翻訳で20-30分
7. ローカル動作確認（Build And Run なら自動・手動なら `python -m http.server` 等の簡易サーバ経由でブラウザから開く）

**itch.io 公開**：
1. itch.io アカウント作成（無料）
2. **+ Upload new project** → Kind of project = **HTML** を選択
3. ビルド出力フォルダ全体（`index.html` 含む）を zip 化してアップロード
4. アップロードしたファイルに「This file will be played in the browser」をチェック
5. Embed options → Viewport **1920×1080**・Fullscreen button ON
6. Visibility = **Restricted**（限定公開URLを友人・審査員に共有）
7. Save & view page → URL を取得

**フィードバック反映**：itch.io は URL を変えずに新ビルドを差し替えられるので、共有先には自動で最新版が届く。

**動作環境（2026-06-04 更新・itch.io ページ説明文にも転記必須）**：
- **PC ブラウザ専用**（Chrome / Edge / Firefox / Safari いずれも可）
- **推奨解像度 1920×1080 以上**（Canvas 1920×1080 固定・2026-06-04 拡大）
- **入力**：マウス＋キーボード（ホバー操作を含む UI 設計）
- **スマホ・タブレットは動作対象外**（タッチでホバーが発生しないため一部機能が使えない・GUI スケーリングも未対応）
- itch.io ページ説明文には**冒頭**にこの注記を明記し、試遊者の期待値ずれを避ける

itch.io ページ説明文の冒頭テンプレ：
> **動作環境：PC ブラウザ推奨（Chrome/Edge/Firefox/Safari・解像度 1920×1080 以上）。スマホ・タブレットは動作対象外です。**
