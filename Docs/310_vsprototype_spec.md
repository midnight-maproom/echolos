## 1. 領地マップ画面

### 1.1 目的

USP「複数戦線で取捨選択を迫られる」を **空間的・視覚的** に伝達する。
特に「**バルドゥイン救援に手を伸ばすほど、防衛が薄くなる**」というジレンマを画面1枚で直感的に見せる。

### 1.2 マップ構造（3列×3層＋本拠地）

```
┌─────────────────────────────────────────────────┐
│  [敵拠点★]    [敵拠点]      [敵拠点]            │  ← 第3層：敵拠点（敵=強）
│  バルドゥイン                                    │
│      │            │              │              │
│  [敵 領]     [敵 領]      [敵 領]                │  ← 第2層：敵領（敵=中強）
│      │            │              │              │
│  [自 領]     [自 領]      [自 領]                │  ← 第1層：自領（敵=中・初期支配）
│      └────────┬───┴──────────────┘              │
│             [本拠地]                             │  ← 王宮（R7 ボス戦の舞台）
└─────────────────────────────────────────────────┘
        左列            中列            右列
```

- **9マス＋本拠地**：3列（左／中／右）× 3層（敵拠点／敵領／自領）＋ 本拠地1つ。
- **戦線差別化なし**：3列の自領・敵領・敵拠点はすべて同パラメータ。
- **左列の敵拠点はバルドゥインが籠城する特殊扱い拠点**（`IsBalduinStronghold=true`）。攻略でブリジット救出。プレイヤー画面上は他列と同じ「敵拠点 左」として表示し、特別性は専用アイコン（`node_balduin`）で伝える。
- **本拠地**：通常時は配置不可。自領が陥落すると本拠地に直接侵攻が発生（§1.5）／R7 でボス戦の舞台。

### 1.3 領地・拠点データモデル

各マスは種別（本拠地／自領／敵領／敵拠点）と列・層の位置を不変属性として持ち、戦況に応じて以下の動的状態を遷移する。マップ全体は 3 列 × 3 層 ＋ 本拠地 1 つの 10 マス固定。

| 状態軸 | 持つマス | 遷移の意味 |
|---|---|---|
| 制圧 / 非制圧 | 敵領 / 敵拠点 | 攻め込み戦勝利で制圧。占領済みマスで奪還戦に負けると非制圧へ戻る（取り戻され・§1.4.2） |
| 健在 / 陥落 | 自領 | 防衛戦敗北＋プレイヤー未配置で陥落。陥落後は取り戻し戦の対象（§1.5） |
| 配置ユニット | 全マス | 上限 6 体（前列 3 ＋ 後列 3）。配置・解除・全クリアの 3 操作 |
| 敵編成 | 戦線が立つマス | ラウンド開始時にセット／ラウンド末にクリア |
| BattleMode | 全マス | このラウンドの役割（戦線外 / 自陣最前線 / 敵陣最前線 / R7 ボス）。詳細は §1.4.2 |

左列の敵拠点はバルドゥイン籠城拠点として識別フラグを持ち、攻略でブリジット救出（§2）。フラグはマップ初期化時にセットされ、メタ強化で永続解禁後は外れる（[330 §8](330_vsprototype_storyplot.md)）。

**設計判断**：
- 状態変更は API メソッド経由のみ。`public` フィールド直公開は禁止（[900 §1.3](900_development_rules.md)）
- 状態変化時にイベント通知発火＝ UI 連動忘れを構造的に防ぐ
- マスの種別（不変属性）と BattleMode（動的属性）を直交させ、「敵不在マスが自動勝利で誤 Capture される」ような暗黙副作用を構造的に防止する

### 1.4 配置・戦闘ルール

- **配置先**：プレイヤーは手駒を **任意のマスに 6 スロットまで** 配置可能。
  - 自領 = 防衛戦
  - 敵領 = 反転攻勢戦
  - 敵拠点 = 攻略戦（同列の敵領を制圧後に解放）
  - 本拠地 = R1-R6 は常時配置可（自領陥落時の本拠地戦備え）／**R7 ボス戦は本拠地のみ配置可**（他マスへの配置は禁止＋既存配置はラウンド開始時に自動回収）
- **戦闘解決**：マス種別によるルール差はなく、編成同士の対決を共通解決器で処理。
- **敵編成**：強度プールからのランダム抽選で決定。詳細は §1.4.1。
- **R7 本拠地ボス**：固定編成（皇太子＋取り巻き 5 体）。プール抽選の対象外（§1.6）。

### 1.4.1 敵編成プール仕様

**狙い**：ローグライトとして「ラン毎に違う編成」を出しつつ、強度カーブを段階的に上げ、地形（列ごと火/水/中立 × 層別強度）による戦術的差別化を生む。場所（自領／敵領／敵拠点）の難度差はステータス係数・編成差し替えではなく **地形強度による DEF 補正**（[320 §5](320_vsprototype_combat_spec.md)）に集約する。

**強度カーブ**：強度プールごとに出現可能なユニット ID とそれぞれの Lv を変える 2 軸構造。

- **プール軸**：強度別に「出現するユニット ID 集合」が増減する（例：全体魔法など特殊な敵は弱プールに出さず、強プールから登場）。同一 ID を複数プールが共有してよい
- **Lv 軸**：敵味方共通の Level（1〜3）で同 ID の強度差を表現。プレイヤー側の強化ペース（§1.11.5）と敵プールの Lv が同じ軸に乗る

**R 別配置体数想定**：

| R 帯 | プール | 味方手駒（累計） | 1 戦線あたり配置体数 | 敵編成体数 |
|---|---|---|---|---|
| R1-R2 | 弱 | 8〜10 体 | 2-3 体 | 3 体 |
| R3-R4 | 中 | 12〜14 体 | 4-5 体 | 4 体 |
| R5-R6 | 強 | 16〜18 体 | 5-6 体 | 6 体 |

#### 構造

最終敵編成は「プール抽選（R 帯）× 各エントリの Lv 指定 × 地形追従（属性タンク選定）× ランダム抽選」で決まり、戦闘解決時にさらに層別地形強度（自領=Light / 敵領=Medium / 敵拠点=Heavy）による DEF 環境項が両陣営に適用される（[320 §5](320_vsprototype_combat_spec.md)）。

| 軸 | 単位 | 仕様 |
|---|---|---|
| プール | ラウンド帯 | R1-2=弱／R3-4=中／R5-6=強。**出現可能なユニット ID とそれぞれの Lv** の集合として定義 |
| Lv 指定 | エントリごと | プールの各エントリが (UnitId, Level) を持つ。同一プール内で異なる Lv が混在しうる |
| 地形追従 | 列地形（火/水/中立） | 火有利列＝火属性タンク含む編成／水有利列＝水属性タンク含む編成／中立列＝属性不問（[320 §5.4](320_vsprototype_combat_spec.md)） |
| ランダム抽選 | ラウンド開始時 | 各マスの編成をプールから独立抽選（1 周目から全ランダム）。乱数源は決定論テスト可能な形で外部から注入 |

1 周目の優しさはドラフト枠保護（§1.11.4）と味方初期固定編成で担保するため、敵編成プールには「1 周目フラグ」を持たない。場所別の難度差は地形強度の環境項（[320 §5.1](320_vsprototype_combat_spec.md)）に一本化、R 進行による強度カーブは Lv 軸に集約する。

#### 強度プール内訳（Required ＋ Random 二段構造）

各プールは「**確定枠（Required）**＋**抽選枠（Random）**」の二段構造を持つ。Required は候補グループから 1 体を抽選して必ず含める枠（例：タンク候補から 1 体）、Random は残数を重複なく抽選する枠。「タンクなし編成」のような明らかな弱編成を構造的に排除し、編成の強さ下限を担保する。

| プール | 確定枠 (Required) | 抽選枠 (Random) | 体数（自領／敵領／敵拠点） | 組み合わせ |
|---|---|---|---|---|
| 弱 | なし | Lv1 × 5 体（剣士／水剣士／弓／水弓／司祭） | 自領 R1-2: 2 | 5C2 = **10 通り** |
| 中 | タンク Lv1（火／水）から 1 | Lv1 × 6 体（弱5 ＋ 聖盾） | 自領 R3-5: 1+2 = 3／敵領: 1+3 = 4 | 6C3 × 2 = **40 通り** |
| 強 | タンク Lv1（火／水）から 1 ＋ バッファ Lv1（火／水）から 1 | Lv1 × 6 体（中と同候補） | 自領 R6 ／ 敵拠点 ともに 2+3 = 5（完全一致） | 6C3 × 2 × 2 = **80 通り** |

プロト範囲ではレア枠相当（焦熱の大魔導士／焔影の暗殺者／癒水の巫女／水鏡の幻盾兵／暁光の大魔導士）はこの 3 プールには登場せず、R7 ラスボス編成（§1.6.1）でのみ使用する（焦熱の大魔導士のみ採用）。**中ボスは VSプロト範囲では実装しない**（R6 マスは強プール抽選のまま運用）。

地形強度別の DEF 補正値は [320 §5.1](320_vsprototype_combat_spec.md) を参照。

#### 並び順優先度（SortOrder）

抽選後の SlotIndex はユニット固有の SortOrder 昇順で一意に決定される（同 SortOrder 内は抽選順そのまま＝決定論）。「弓兵が最前列に出る」事故を排除し、毎ランダム抽選でも編成の戦術的整合性を保証する。

| SortOrder | 役割 | 該当ユニット |
|---|---|---|
| 1 | メインタンク | 火タンク／水タンク／水鏡の幻盾兵 |
| 2 | サブタンク | 光の騎士 |
| 3 | 近接アタッカー | 剣士／水剣士／焔影の暗殺者／王女／ブリジット |
| 4 | 後衛支援（ATK 0） | 火バッファ／水バッファ／癒水の巫女／光の司祭 |
| 5 | 遠隔（最後尾で配置 ATK 補正 max） | 火魔導／火弓／水弓／暁光の大魔導士 |

味方陣営は手動配置なので SortOrder は参照されない（敵編成生成時のみ使用）。

#### マス種別 → プールマッピング

| 戦闘種別 | 適用プール |
|---|---|
| 自領（防衛戦） | R 帯切替（R1-2 弱／R3-5 中／R6 強）。**防衛だけで徐々に厳しくなる圧**で反転攻勢の判断を迫る。R7 は本拠地ボス戦のみで自領戦は発生しない |
| 敵領（攻め込み戦） | 常に中（攻める側のマス強度は固定） |
| 敵拠点（攻め込み戦） | 常に強（同上） |
| 本拠地戦 | R 帯切替（§1.5・連戦なし・自領防衛と同プール） |

ラウンド開始時に全マスの編成を一括決定し、同一ラウンド内では編成変動なし。R7 ラスボスマスは固定編成（プール抽選対象外・§1.6.1）。

#### 進軍ルール

- バルドゥイン救出には左列で **敵領 → 敵拠点** の順に勝利する必要がある（最速 R1〜R2）
- 敵領で勝利＝そのマス制圧。以降そのマスは自陣の最前線になる（§1.4.2）
- 制圧した敵領の奥にある敵拠点に次ラウンドから配置可能

#### 1.4.2 戦線概念（自陣最前線／敵陣最前線／奪還戦／取り戻され）

列ごとに独立した「戦線」を持ち、占領状態で**自陣の最前線が前進**する設計。プレイヤーには「いつ反転攻勢を仕掛けるか」「占領した拠点を維持する兵力をどう割くか」の戦略判断を迫る。

##### 列ごとの最前線判定

| 列の状態 | 自陣最前線（防衛戦／奪還戦／取り戻し戦の対象） | 敵陣最前線（攻め込み戦の対象） |
|---|---|---|
| 何も占領なし・自領健在 | 自領（layer=1）＝防衛戦 | 敵領（layer=2） |
| 敵領占領済み・自領健在 | 敵領（layer=2）＝奪還戦 | 敵拠点（layer=3） |
| 敵領＋敵拠点占領済み・自領健在 | **なし**（列の戦線終了） | **なし** |
| 自領陥落（IsFallen） | **自領（layer=1）＝取り戻し戦**（§1.5） | **なし**（陥落自領で隣接性切れ・取り戻すまで先には進めない） |

「領地を奪い合うゲーム」として地理的隣接性を素直に反映する。自領陥落でも敵領／敵拠点に攻め込めるという緩い設計は採らない＝取り戻し戦に勝って自領を回復すれば、次ラウンドから隣接した敵領にも攻め込めるようになる（戦線が前進する）。ラウンド開始時に列 0〜2 それぞれで最前線を判定し、その最前線マスにのみ敵編成をセットする。

##### BattleMode（マスのラウンド役割を明示する一級概念）

各マスは BattleMode でこのラウンドの役割を明示的に保持する。マス種別（不変属性：自領／敵領／敵拠点／本拠地）と直交する動的属性。

| BattleMode | 意味 | セットされる条件 |
|---|---|---|
| None | 戦線外。戦闘予定なし＋状態遷移なし | 戦線終了マス（列全制圧の自領／敵領／敵拠点）など |
| Defense | 自陣最前線（防衛戦／奪還戦／取り戻し戦） | 自陣最前線として返ったマス |
| Attack | 敵陣最前線（攻め込み戦） | 敵陣最前線として返ったマス |
| Boss | R7 本拠地ボス戦 | R7 の本拠地 |

ラウンド戦闘解決は BattleMode != None のマスのみを対象にし、状態遷移は BattleMode 別の宣言的ルールで分岐する。これにより「敵編成 0 マスが自動勝利で誤 Capture される」ような暗黙副作用が構造的に防止される。

##### 攻め優先の順序保証

R1-R6 のラウンド戦闘は **攻め込み戦を先に解決し、攻め込み成功で前線が後退した列の防衛戦を不発化したうえで、残った防衛戦を解決する**。

- 敵領を占領した列で健在自領が防衛戦予定 → 防衛戦免除（敵領と接しなくなった自領は前線から外れる）
- 敵拠点を占領した列で占領済み敵領が奪還戦予定 → 奪還戦免除（列の戦線完全終了）
- 陥落自領の取り戻し戦は敵領／敵拠点との接続と独立した戦闘軸＝免除対象外

本来は「侵攻戦を解決してから防衛戦の発生を再判定する」のが筋。プロト範囲では同時解決の体裁で内部に吸収しているが、フル版では明示的なフェーズ分割を検討する。

##### 敵編成プールの使い分け

| 戦闘種別 | プール | 体数 |
|---|---|---|
| 自陣最前線（自領防衛戦／占領済み敵領の奪還戦／陥落自領の取り戻し戦） | **R 帯切替**（R1-2 弱／R3-5 中／R6 強） | R1-2: 2／R3-5: 3／R6: 5（敵拠点と完全一致） |
| 敵陣最前線（未制圧敵領への攻め込み戦） | **中**（固定） | 4 |
| 敵陣最前線（未制圧敵拠点への攻め込み戦） | **強**（固定） | 5 |
| 本拠地戦（ラウンド末に陥落自領が残っていれば 1 回・§1.5） | **R 帯切替**（同上） | 自領防衛と同じ（R1-2: 2／R3-5: 3／R6: 5） |

**攻めてくる敵は R 帯ベース**（マスがどこかは関係しない）。「防衛だけで徐々に厳しくなる」圧は自領／奪還戦の敵領／取り戻し戦の陥落自領／本拠地戦すべて同一プールで担保。

##### 取り戻されルール

奪還戦の結果で占領済み敵領／敵拠点の状態が変化する：

| 状況 | 結果 |
|---|---|
| 占領済みマスに配置 → 戦闘勝利 | 制圧維持 |
| 占領済みマスに配置 → 戦闘敗北 | **取り戻され**（制圧解除＝敵領／敵拠点に戻る） |
| 占領済みマスに**未配置** | **取り戻され**（自動・戦線後退） |
| 陥落自領に配置 → 戦闘勝利 | **取り戻し成功**（陥落解除＝健在自領に戻る） |
| 陥落自領に配置 → 戦闘敗北 | 陥落維持 |
| 陥落自領に**未配置** | 陥落維持（取り戻されない・本拠地戦のリスクが残る・§1.5） |

「占領した拠点は攻めて取ったあと放置で永久安全」にはならない。維持にも兵力が必要＝退化戦略（防衛だけで篭る）を構造的に抑止する。

##### 配置の継続性ルール

ラウンド間の配置引き継ぎは以下のルールで動く：

| 戦闘結果 | マスの状態 | 配置ユニット |
|---|---|---|
| 防衛勝利／奪還勝利／取り戻し勝利 | マス健在 or 制圧維持 | **配置維持**（次ラウンドへ引き継ぎ） |
| 健在自領で敗北 | 自領陥落 | **マスから外す**（手持ちに戻り再配置可能に） |
| 占領済み敵領で奪還戦敗北 | 取り戻され | **マスから外す**（同上） |
| 攻め込み戦で敗北（未制圧敵領／敵拠点） | 制圧失敗のみ・状態遷移なし | **配置維持** |
| 戦闘不能 | — | **配置維持**（負傷離脱ではなく戦線復帰可） |
| 本拠地配置 | — | **配置維持**（本拠地戦予告に備えた配置を毎ラウンド再入力しなくていい・§1.5） |

##### HP の扱い

VSプロトでは **戦闘間の HP 引き継ぎは仕様外**＝戦闘中以外は常に HP 満タンを保つ。ラウンド解決末尾で味方全員の HP を最大値に戻す。負傷離脱や HP 引き継ぎといった戦闘間継続性は本実装フェーズ送り。

##### 配置可否

戦線概念に整合して、配置可否は **「戦闘が発生し得るマスか」だけで判定** する純粋な指標にする：

| マス種別 | 配置可能条件 |
|---|---|
| 自領 | **常に可能**（健在＝防衛戦／陥落＝取り戻し戦の進攻配置） |
| 敵領 | BattleMode != None（戦線外への新規配置は地理的整合性で不可） |
| 敵拠点 | 同列の敵領が制圧済（攻略順序依存）＋ BattleMode != None |
| 本拠地 | 常に可能。R7 は本拠地のみ可（他マス禁止） |

##### 戦線外配置の自動回収

ラウンド開始時に配置不可マスのユニットは自動的に手駒に戻す。手駒は所持の全集合なので、配置から外せば自動的に「手持ち」に復帰する。

| 発生経路 | 例 |
|---|---|
| 攻略順序違反 | 敵拠点攻め込み敗北＋同列敵領奪還戦敗北で敵領取り戻され → 次 R 開始時に敵拠点は攻略順序ガード違反で配置不可 |
| 戦線終了 | 敵領＋敵拠点とも占領済みで列戦線完全終了＝戦線外化した占領済み敵拠点 |
| 隣接性喪失 | 敵領占領中に自領陥落で敵領が戦線外化 |
| R7 ボス戦 | 本拠地以外のマスに配置されていたユニット |

「配置できないマスに置きっぱなしになる」現象を構造的に防ぐ＝プレイヤーから見て「戦線外にユニットを置く意味がない」設計が一貫する。

### 1.5 自領陥落と本拠地防衛戦

「**陥落自領を取り戻すチャンスを与え、ラウンド末に残っていれば本拠地戦が 1 回だけ発生**」する設計。

##### 自領陥落の判定

健在自領で **戦闘敗北 or プレイヤー未配置** が発生すると陥落。陥落してもそのラウンドの本拠地戦は発生しない（猶予あり）。

##### 取り戻し戦

陥落自領は次ラウンド以降「敵側に占領された状態」として扱う：

- ラウンド開始時に陥落自領にも敵編成セット（R 帯切替プール・自領防衛と同一）
- プレイヤーが配置 → 取り戻し戦が発生
- 勝利 → 取り戻し成功（陥落解除＝健在自領に戻る）
- 敗北 or 未配置 → 陥落維持

##### 本拠地戦

発火判定はラウンド開始時とラウンド末の二段：

| ラウンド開始時 | ラウンド末 | 本拠地戦 |
|---|---|---|
| 陥落自領なし | 任意 | **不発**（新規陥落しても次ラウンドまで猶予） |
| 陥落自領あり | 全自領取り戻し済み | **不発**（完全リカバリ・本拠地未配置でも安全） |
| 陥落自領あり | 陥落自領が 1 つ以上残る | **発生**（1 回のみ・連戦なし） |

- **敵編成**：R 帯切替プール（R1-2 弱／R3-5 中／R6 強・自領防衛と同一）。体数も同じ
- **負ければ即ラン敗北**（Defeat エンディング・§1.7）
- **予告 UI**：ラウンド開始時に陥落自領があれば本拠地に予告編成をセット。プレイヤーは本拠地マスをホバーすると攻めてくる敵を事前に確認でき、本拠地配置の必要性を判断できる。本拠地戦の実編成は予告をそのまま使う（決定論性保証）。全自領取り戻し成功時は予告編成をクリア

##### 設計意図

- 「**うっかり 1 ヶ所負けたら即終了**」を避け、リカバリの選択肢（取り戻し戦／反転攻勢続行／本拠地に集中）を与える
- 一方で「自領全捨てて本拠地に引きこもる」退化戦略は本拠地戦敗北で即終了するため残らない
- 全自領取り戻し成功なら本拠地戦不発＝完全リカバリで報われる設計

### 1.6 R7 本拠地ボス戦

R7 では他戦線が無関係になり、本拠地に皇太子（ボス）が直接襲来する演出。プレイヤーは **本拠地に 6 ユニット配置** してボスを迎撃する。

- 他のマスには配置不可（演出として封鎖・配置済みユニットはラウンド開始時に自動回収）
- R6 までに自領陥落があっても、R7 では本拠地ボス戦のみ。§1.5 の本拠地防衛戦は R7 では発生しない（演出優先）
- **R7 開幕演出は二分岐**：ペンダント未気づき時は **闇皇太子（必敗版）** ／気づき済は **通常皇太子** で編成が切り替わる。詳細は [330 §6.4](330_vsprototype_storyplot.md)

#### 1.6.1 ラスボス編成（通常版・A-c2 経路）

固定編成（プール抽選対象外）。編成意図は以下の対戦構造：

| Slot | ユニット | Lv | 意図 |

| 0（最前） | 帝国 水の大盾兵 | 2 | 専守＋被ダメ -30% でプレイヤー攻撃を吸う |
| 1 | 皇太子 | 1（固定） | ATK 最高値＝炎の鼓舞師の ATK バフ対象として確実に乗る |

| 3 | 帝国 光の司祭 | 2 | 後衛から回復 |
| 4 | 帝国 焦熱の大魔導士 | 2 | 後衛から範囲攻撃（SearingWound 蓄積）。**プロト 3 プールには登場せずラスボスで初登場** |
| 5（最後尾）| 帝国 光の騎士 | 2 | プレイヤー弓兵から後衛を守るサブタンク（編成順固定で最後尾配置） |

- Lv 強化なし（Lv1 固定）
- 2 Waza 交互発動（CD ずらしで毎ターン何かを撃つ）：単体攻撃＋ DEF デバフ／全体攻撃

#### 1.6.2 ラスボス編成（必敗版・A-c1 経路）

通常版（§1.6.1）と **同じ取り巻き 5 体構成・Slot 1 の皇太子だけを闇皇太子に置換** する形（編成見栄え優先）。取り巻きを先に倒しても闇皇太子の攻撃力が累積するため構造的に詰む設計。

闇皇太子本体の性能は [320 §4.8](320_vsprototype_combat_spec.md) を参照。詰みを保証する 3 要素：

- 実質無敵 HP/DEF（物理・魔法とも実質ゼロダメ）
- 毎ターン確定発動の全体攻撃＋自己 ATK 永続加算スタック（解除不可）
- スタック累積で T4 までに味方全滅級ダメージに到達

### 1.7 ラン勝敗判定とエンディング 2 分岐

ラン結果は **Defeat / True** の 2 種類で確定する。詳細仕様・分岐ロジック・3 分割 Defeat 演出の振り分け条件は [330 §6](330_vsprototype_storyplot.md) を SSoT とする。要点：

- 本拠地戦敗北（陥落自領が残ったまま）→ Defeat
- R7 必敗版で進入（A-c1 経路）→ Defeat
- R7 通常版で進入し戦闘敗北（A-c2 経路）→ Defeat
- R7 通常版で進入し戦闘勝利（A-c2 経路のみ可能）→ True

**R6 = SLG ゲームクリア体裁**：R1〜R6 防衛成功で SLG としては「ゲームクリア」達成扱い。R7 皇太子撃破はストーリー的やりこみ・おまけとして位置付ける（[330 §6.1](330_vsprototype_storyplot.md)）。

### 1.8 表示仕様

PC 16:9 横長前提。マップ画面は 3 列分割で横画面を活用する。

#### 画面レイアウト（マップ画面）

| 領域 | 内容 |
|---|---|
| **左 20% 王国軍パネル** | 表示専用。配置先マス別グルーピング（「未配置 → 本拠地 → 自領 左/中/右 → 敵領 左/中/右 → 敵拠点 左/中/右」の 11 グループ固定順）で各見出しに「N/6」配置済人数を表示し、空グループも見出しを残して配置不在マスを一目可視化。各行はアイコン＋名前＋簡易ステータス（Lv／HP／ATK／属性 1 文字（色付き）／攻撃種別／役割タグ）。グループ内ソートは固有→Normal→Rare 順。**行ホバーでユニット説明ツールチップ**（属性・攻撃種別・役割タグ・フルステ＋ Description 本文）を表示し、ドラフト時の説明を戦略マップから何度でも参照できる |
| **中央 55% マップ本体** | マップ背景画像＋拠点アイコン 10 個（9 マス＋本拠地） |
| **右 25% 情報パネル** | ヘッダー（R/7）／戦闘結果サマリ／ガイド（Flash メッセージ）／**戦線状況**／**属性シナジー Tips**／進行ボタン |

右パネル進行ボタンは Phase 別に切替：

- **内政フェーズ中**：行動力表示＋「召集」「ユニット強化」「内政終了」3 ボタン
- **配置フェーズ中**：「戦闘実行」「次のラウンドへ」「メタ拠点へ」（状態遷移で切替）

右パネル下半分の常時情報：

- **戦線状況**：戦線が立っているマスのみ「本拠地→自領→敵領→敵拠点」順で `[!]/[△]/[○]` 記号＋マップバッジ枠色と同配色で列挙。マップ上の色バッジを文字でも再表現し「どこに何体足りないか」を一覧化
- **属性シナジー Tips**：火／水／光の 3 属性を 1 行ずつ説明。数値は出さず「2 体から発動・人数で強化・水のみ 4 体節目で質的追加」が伝わる粒度に抑える（試遊者の暗記負荷回避）

#### マップ描画

- **背景＋拠点アイコン**：マップ背景画像を中央パネルに描画し、各マスに**所属（現在どちら陣営が保持しているか）対応**の拠点アイコンを透過配置する
- **アイコンキー（所属別）**：
  - 自軍所属 → 自領／占領済み敵拠点／本拠地のそれぞれ専用画像
  - 敵軍所属 → 自領陥落＋未占領敵領で共用／未占領敵拠点／未解放バルドゥイン拠点（左列専用）の画像
  - 占領／陥落で色加工はせず画像自体が切り替わる。左列敵拠点も解放後は通常の自軍砦アイコンに統一。マスラベルのテキストは他列と同じ「敵拠点 左／中／右」で統一し、特別性は専用アイコンと B-d 救援演出側で伝える
- **ノード配置**：マップ背景画像内に描かれた円位置に合わせてアイコンを内接配置。拠点（城型）と野営地（camp 型）でサイズを変える
- **マス背景色／枠／ラベル／配置スロット小アイコンは持たない**：マップ背景画像と拠点アイコンの絵だけで種別・状態を伝える。配置済みユニットの可視化は左ペイン配置先別グルーピング＋マス人数バッジ＋拠点ホバーツールチップで吸収

#### マス人数バッジ

各マス下端中央に「味方-敵」形式の数字を小バッジで描画。**枠色で BattleMode と配置状況を端的に伝える**＝数字と色だけで「攻撃予告」「劣勢／安全」を一目で判断できる。敵編成の中身はホバーツールチップで確認する。

| BattleMode | 未配置 | 劣勢（味方 < 敵） | 安全（味方 ≧ 敵） |
|---|---|---|---|
| Defense（自陣最前線：防衛／取り戻し／奪還）／ Boss（R7 本拠地ボス） | 赤枠（確定敗北） | 黄枠 | 水色枠 |
| Attack（敵陣最前線：攻め込み） | 灰枠（攻めず） | 橙枠（劣勢警告） | 青枠 |
| None（戦線外） | 灰枠 | 灰枠 | 灰枠 |

判定境界は単純頭数比較。**戦線外の敵領／敵拠点はバッジ非表示**（情報ノイズ回避）。戦線外の自領／本拠地は 0-0 でも灰枠で表示（配置可能なのでレイアウト一定性優先）。

#### 拠点ホバーツールチップ

マウスがマス上にあるとき敵編成＋味方配置の 2 セクションを表示。マス種別で自動切替：敵領／敵拠点は両セクション、自領／本拠地は味方のみ（敵編成ゼロのため）。表示形式は兵種名×N グルーピング＋前列／後列件数サマリ。味方ゼロは「（未配置）」と明示。両セクションが空のマスでは表示しない。

#### 進攻ルート可視化

マップ背景画像内の道で表現する（独立した接続線描画は持たない）。

#### 1.8.1 配置動線（統合モーダル）

配置と解除はマスクリックから開く統合モーダルに一本化する。左パネル王国軍リストは表示専用（クリック操作なし）。

#### 1.8.2 戦闘画面

戦闘実行ボタンを押すと戦闘再生フェーズに遷移し、ラウンドの全戦闘（マス順 → 本拠地戦 or R7 ボス戦）を順次連鎖する。再生完了で配置フェーズに復帰＝結果サマリを表示する。

| 領域 | 内容 |
|---|---|
| **上部ヘッダ** | マスタイトル（例：「自領 中央」「敵拠点 左」「本拠地 ボス戦」）＋「R/7」＋「戦闘 i/N」 |
| **中央アリーナ** | 6 vs 6 の前列 3 ＋後列 3 対峙構図（アイコンのみ）。敵側は左右反転で対峙化。死亡ユニットは暗色化。**ユニット情報（HP・名前）は下部ステータスリストに集約**してスロット内重複を避ける |

**戦場背景**：草原系の背景画像をアリーナ領域だけに ScaleAndCrop 描画。全画面ベースは事前に単色塗りされており、パネル間隙間・アセット未配置はそのまま単色が見える＝フォールバック自然動作。アリーナ領域は約 2.77:1 の横長。地平線を画像上 25〜35% に配置すると、ScaleAndCrop でアリーナ内に地平線が来てキャラ前列が地面に乗る絵になる。

**スロット背景塗りは持たない**：透過キャラ画像を戦場背景の上に直接描画する。敵味方区別は位置（左右）＋反転＋下部ステータスリストで十分視認可。死亡視認は暗色化。

**キャラレイアウト**：ユニット画像が正方形のため、行間を意図的に詰めて隣接行を重ねる＋セルからはみ出すサイズに拡張し、後列が前面に重なる順で描画して奥行き感を出す。

**擬似 3D 奥行き演出**：縦並び 3 レーンを「奥→手前」の遠近表現にする。両陣営とも奥が中央軸寄り／手前が外側＝消失点が画面中央奥に来る遠近表現。中心保持で表示サイズだけを調整する純粋な視覚演出で、当たり判定・スロットインデックスには影響しない。

**横方向の領域配分**：アリーナを左右 2 分割する中央余白を確保し両陣営の前列が無人地帯を挟んで対峙する空間を作る。各陣営内は前列と後列を意図的に重ね、前列が後列を遮る奥行き演出にする。「味方と敵の間は広く・前後列の間は狭く」の構成。

**状態異常バッジ**：各ユニット画像の左上（敵側は右上）に最大 6 個の小バッジを横並びで重畳表示。前列がアリーナ上端からはみ出さないようやや下寄りに配置。上限超過時は最後に「+N」バッジで畳む。

| 状態 | 描画 |
|---|---|
| 画像あり | 状態異常別アイコン画像を内接描画 |
| 画像なし（既定） | 半透明黒背景＋カテゴリ色枠＋英字 2 文字。スタック 2 以上は右下に小数字 |

**カテゴリ色枠**：効果の寿命と解除可否で分類し色分けする。

| カテゴリ | 色 | 例 |
|---|---|---|
| Cleansable | 赤枠 | 燃焼／凍結／熱傷／麻痺（Cleanse で剥がせる状態異常） |
| Persistent | 青枠 | パッシブ・オーラ・シナジー（永続バフ・解除不可 or 一部解除可） |
| Triggered  | 黄枠 | 一時バフ・デバフ（残ターン経過で自然消滅） |

**バッジホバーツールチップ**：バッジ上にマウスホバーで「効果名（カテゴリ）／発生源／スタック数／残ターン or 永続」を直下に表示。画面外はみ出し時は上に出す。モーダル中はホバー反応抑止。

**シナジー発動バッジ**：アリーナ左上に火／水／光 3 種のバッジを重畳表示。味方陣営の属性体数からシナジー段階を直接参照（味方陣営限定発動）。

| 状態 | 描画 |
|---|---|
| 画像あり | 属性別シナジーアイコン＋右下に段階数オーバーレイ |
| 画像なし（既定） | 属性カラー帯＋「F2／W4／L0」のような英字＋段階数文字フォールバック |
| 未発動（0-1 体） | 文字フォールバックは灰色帯／画像は彩度を落とす |

**シナジーホバーツールチップ**：バッジ上にマウスホバーで直下に Tier 効果の要約を表示。モーダル表示中はホバー反応を抑止。

**スコープ外**：エフェクト演出／装飾枠／リーダースキル・撤退／ターゲット線。

### 1.9 ビジュアル品質方針（戦闘画面）

戦闘画面は「キャラを並べただけ」ではなく「戦場に立つ」表現を最低限満たす方針。エフェクト・モーション無しでも、背景＋透過配置＋構図だけで雰囲気が一段上がる絵を目指す。

**含む**：
- 戦場背景画像 1 枚共通（草原系）。アリーナ領域に背景描画
- ちびキャラ透過化：透明背景で生成し、戦場背景の上に重ねて配置
- 配置レイアウト：前列 3 ＋後列 3 で両軍が向かい合う対峙構図

**スコープ外（試遊版送り）**：
- 戦場背景の複数枚化（自領用・敵領用などの差別化）
- エフェクト演出のリッチ化（Particle・Spine・フレームアニメ等）
- UI パネルの装飾過多
- 戦闘中アニメーション（攻撃モーション・ダメージリアクション等）

### 1.11 内政フェーズ

「**メタ強化＝間接強化／ラン内＝直接強化**」の二層構造でラン内成長サイクルを成立させる。**兵種ラインナップ詳細**（ID・ステータス・技構成）は [320 §4.8](320_vsprototype_combat_spec.md) ＋実装側 Roster を SSoT とする。

#### 1.11.1 1 ラウンドの流れ

```
R1〜R6：
  内政フェーズ開始時：自動加入 1 体（§1.11.4）
  内政フェーズ（行動力 2 を消費・繰越なし）
    → 配置フェーズ（マスを選び、6 スロット中の前列 3 ／後列 3 に配置）
    → 戦闘実行（全マス＋本拠地戦が必要なら 1 回）
    → 次ラウンドへ
R7：ボス戦のみ（内政フェーズなし・本拠地に直接配置）
```

ラウンドごとの手駒増加は **自動加入 1 体 ＋召集 1 体 ＝計 2 体**（R6 終了時に初期 6 体 ＋ 12 体 ＝ 18 体）。1 戦線あたりの最大配置数は R1 で 2-3 / R3-4 で 4-5 / R6 で 6 体まで段階的に増える。

#### 1.11.2 内政コマンド

| コマンド | コスト | 効果 | 制約 |
|---|---|---|---|
| **召集** | 行動力 1 | 3 択ドラフトでユニット 1 体を手駒に追加 | 同一ラウンド 1 回まで |
| **ユニット強化** | 行動力 1 | 個別ユニットを 1 体選び、3 択中の 1 つで Lv +1（基礎ステ強化） | 各ユニット Lv 上限 3／**同一ラウンド複数回可**（行動力の続く限り） |

オミット項目（VSプロトでは持たない）：

- **拠点強化**：VSプロトはマス単一統一モデル＝戦線拠点 BaseLevel 概念なし
- **偵察コマンド**：常時偵察状態に置き換え（敵編成は常に可視化）
- **治療／建設／資材購入／イベント**：フル版送り

#### 1.11.3 行動力

- 毎ラウンド 2（繰越なし）
- メタ強化「行動力 +1」で +1 → 3 に解放（§3.2）
- **召集**は同一ラウンド 1 回まで（単調な極振り防止）／**ユニット強化**は同一ラウンド複数回可（行動力の続く限り）

#### 1.11.4 召集ドラフト＋初期ドラフト＋自動加入

ドラフトプールは **通常 9 体＋レア 5 体**。ブリジット・王女・皇太子は固有ユニット扱いでプールから除外。

**初期ドラフト（2 周目以降）**：ラン開始時に **5 回**（メタ強化「初期所持ユニット +1」Lv 分だけ追加）。プレイヤーが 3 択から 1 体ずつ選ぶ。全枠 Normal プール固定（Rare 抽選なし）。

**初期ドラフト（1 周目限定スキップ）**：1 周目のみ初期ドラフトをスキップし、固定構成を Roster に直接投入する。火・水・光の 3 属性×剣士／タンク／ヒーラーの最小バランス構成で、王女（光属性）と合わせて火 2／水 2／光 2 の全シナジー 2 体段階を成立させる初期編成。プレイヤーが「与えられたユニットの配置を考える」ことに集中できるよう、編成判断ゲームは召集コマンドで提供する設計。

**召集ドラフト**：内政フェーズの「召集」コマンドで 1 回（行動力 1 消費・同一ラウンド 1 回まで）。3 択。**枠ごとに独立して Rare 抽選**する 2 段構成：

1. まずオファー全体で **3%** の「★全 Rare スペシャル」判定。当選すると 3 枠すべてが Rare プールから抽出される
2. 落選時は通常モードに入り、枠ごとに **15%** で独立 Rare 抽選。Rare 枠は Rare プール／Normal 枠は Normal プールから抽出

体感確率：Rare を 1 体以上見る確率は約 40.5%・全枠 Rare（★演出）は約 3.34%（明示 3% ＋偶発 0.34%）。偶発的に全枠 Rare 抽選された場合も明示モードと同じく ★ヘッダ表示（プレイヤー視点では区別しない）。

**3 体から必ず 1 体選択（キャンセル不可・ガチャ的やり直し封じ）**。やり直したい場合は次ラウンドの召集をやり直す。

##### 自動加入

R3-R4 帯で 1 戦線に 4-5 体配置を成立させるため、内政フェーズ開始時に **ランダム自動加入 1 体**が発生し、プレイヤーの手駒に追加される。召集と並存し、ラウンドごとに 2 体ずつ手駒が増える構造。

| 項目 | 仕様 |
|---|---|
| 発火タイミング | 内政フェーズ開始時（自動） |
| 候補抽選順 | ① 手持ち未所持の **Normal 枠**から抽選 ／ ② Normal 全所持済なら手持ち未所持の **Rare 枠**から抽選 ／ ③ 全枠所持済なら全候補（Normal + Rare）からランダム抽選（重複加入） |
| 行動力コスト | なし（無料） |
| プレイヤー通知 | Flash メッセージ |
| ドラフトプール | 召集と同じプールを共有 |

#### 1.11.5 ユニット強化

**ユニット個別の永続強化**。ラン内永続、次ランに引き継がない（メタ強化と区別）。

| 項目 | 仕様 |
|---|---|
| Lv 上限 | Lv3（強化 2 回） |
| 強化対象 | **個別ユニット**。同種ユニットが複数いても A は Lv3／B は Lv1 のような状態が成立する |
| 強化選択 | 各 Unit が固有の **3 択強化選択肢**を持ち、Lv2 で 1 つ・Lv3 で残り 2 択から 1 つ選択。**3 択中 2 つ取得、1 つ取り損ねる**（強化ルートの選択にプレイ判断の重みを持たせる） |
| 強化内容 | 基本ステ上昇＋技の数値強化。**新規技追加・効果激変はしない**（あくまで値を増やすだけ・偵察予測しやすさを優先） |
| 適用方法 | 一般ユニット：内政画面のユニット強化 UI（行動力消費）から選んで強化 |
| マスタボーナス | プロトでは未実装（フル版要素） |

**敵側との対称性**：

- 敵 Unit も同じ 3 択強化選択肢を持つ
- 敵は **順序番号 0/1/2** を裏で持ち、Lv 上昇のたびに先頭から順次適用（Lv2 で番号 0、Lv3 で番号 1 を適用）
- 順序番号 2 は適用されない＝味方の「1 個取り損ね」と完全対称
- 敵側でランダム化はしない（[推測容易性原則](320_vsprototype_combat_spec.md)・偵察ツールチップで Lv 別の効きが予測できる）

**固有ユニット（王女・ブリジット）の扱い**：

- **内政画面の強化 UI 対象外**：「内政画面の動線とは別経路」という位置付け
- **メタ強化画面から強化**（メタ通貨消費・§3.2）。3 択構造は同じ

##### 強化種別

| 種別 | 効果 | 例 |
|---|---|---|
| 基礎ステブースト（ATK/DEF/HP） | 強化対象ユニット本人の基礎値に加算 | ATK+5／DEF+3／HP+20 |
| Waza 強化 | 対象技の Magnitude（バフ量・回復量等）に加算 | 鼓舞 +5・浄化の癒し +1 |
| 永続効果強化 | 対象ユニットの永続パッシブ効果値に加算 | 専守 +10%（被ダメ -30%→-40%） |
| オーラ強化 | 対象オーラのバフ量に加算 | 王家の加護 +2 |

具体的な Upgrade ID／Magnitude／対応兵種は実装側 Roster を SSoT とする。

##### 強化選択肢（Preset）

兵種ごとに 3 択の Preset を持つ。Preset の順序ルール：

- **固有スキル持ち兵種**：固有スキル（順序 0）→ 第 2 ステ（順序 1）→ HP（順序 2）。敵 Lv2 で固有スキル強化、Lv3 でさらに +DEF が適用される
- **固有スキルなし（汎用 Attacker/Tank）**：ATK → DEF → HP の順。敵 Lv2 で ATK+5、Lv3 で +DEF+3
- **HP +20 を順序 2 に置く**：敵に乗らない＝プレイヤーの選択枠としてのみ機能

各兵種への具体的な Preset 割り当ては実装側 Roster を SSoT とする。

**UI での可視化**：左ペイン王国軍リスト・配置モーダル一覧・強化リストの 3 箇所で「Lv」を常時表示（**Lv2 以上は金色アクセントで強調**）。ステ表示は強化適用後の値を使い、強化結果が即座にプレイヤーに見える。**行動力 0 でも強化選択画面までは進める**（試行錯誤を許容）が、最終の「採用」ボタンは行動力 0 で disabled。

#### 1.11.6 前衛 / 後衛・距離・守護

戦闘プロトと同一仕様：

- **配置スロット**：1 マス＝前列 3 ＋後列 3（合計 6）
- **近距離攻撃**：前列のみ攻撃可（後列メレーは無効化）
- **遠距離攻撃**：列制限なし（後列の弓兵・魔法は前列越しに攻撃可）
- **後列むき出し判定**：同列前列スロットが死亡／不在なら遠距離以外でも後列が狙われる
- **守護スキル**：[110](110_combat_spec.md) ／ [320](320_vsprototype_combat_spec.md) の仕様に従う

配置 UI：

- マスクリックで配置統合モーダル展開（§1.8.1）
- **配置引き継ぎ**：R2-R6 開始時、前ラウンドの生存配置（健在自領＋制圧済敵領／敵拠点）を同マス・同スロットへ自動復元する。陥落自領・本拠地・R1・R7 は対象外。復元時に HP は最大値に戻る（戦闘間 HP 引き継ぎは仕様外）
- **役割タグ表示**：ドラフトカード／配置モーダル一覧／強化リストに役割タグ（`[盾/攻]` 等）を表示
- **王国軍／配置モーダル一覧／強化リストのソート**：表示順は「固有ユニット（王女→ブリジット）→ DraftPool Normal index 順（前衛 6 →後衛 6）→ Rare index 順 → その他」。**二次ソートは Lv 降順**（Lv3 → Lv2 → Lv1）

#### 1.11.7 常時偵察（敵編成可視化）

- 各マスのホバーで**敵編成の詳細をツールチップ表示**（PC ホバー前提・[900 §2.1](900_development_rules.md)）
- 偵察コマンド自体は持たない（情報は常に与え、配分判断に集中させる）

#### 1.11.8 メタ強化との関係

| メタ強化項目 | 意味 |
|---|---|
| **王女 Lv 強化**（Lv2 / Lv3 で各 1 回） | §1.11.5 の 3 択選択を実行。プレイ判断の重みあり |
| **ブリジット Lv 強化**（Lv2 / Lv3 で各 1 回） | 同上。R5 までに加入していないと無意味 |
| **行動力 +1**（2→3） | 内政枠の解放。コマンド多用が可能に |
| **初期所持ユニット +1**（最大 3 回） | ラン開始時の初期手駒 +1 |

固有ユニットの強化はすべてメタ強化動線に集約される（内政画面の UI 対象外）。詳細は §3.2。

#### 1.11.9 Phase 設計拡張

ラン開始の流れ：

```
Hub → InitialDraft → InteriorAction(R1) → Run(R1) → InteriorAction(R2) → … → R7 のみ InteriorAction を飛ばして Run へ直接
```

ボス戦（R7）は内政なし。配置・戦闘のみ。

**内政動線一体化**：内政フェーズ中もマップ画面が描画され、マップ配置を有効化したまま右パネルから内政コマンドを実行できる。「召集」「ユニット強化」クリック時のみ専用 GUI が半透明オーバーレイ＋中央モーダルとして上に重なる。モーダル外クリックでサブモードキャンセル。「内政終了」で配置フェーズに遷移し戦闘実行ボタンが解放される。

### 1.12 ユニット命名規則

**Name フィールド**（表示用・日本語）：

| 区分 | パターン | 例 |
|---|---|---|
| 非レア | 属性 + の + 役職 | 炎の双剣士／水の大盾兵／光の司祭 |
| レア | 特殊冠詞 + 役職 | 焦熱の大魔導士／焔影の暗殺者／水鏡の幻盾兵／癒水の巫女／暁光の大魔導士 |
| 固有 | 固有名 | 王女／ブリジット |

属性表記は **火属性 → 「炎の」** ／ **水属性 → 「水の」** ／ **光属性 → 「光の」** に統一。役職は支援系を「師」（鼓舞師／護術師）、戦闘者を「士／兵／者」（剣士／大盾兵／暗殺者）で使い分け。

**Id フィールド**（識別子・英字）：

- パターン：`{属性}_{役職}` の 2 要素（例：`fire_swordsman`）
- 属性プレフィックス（`fire_` / `water_` / `light_`）は揃える＝ソート時に同属性が固まる
- 役職部分は職能名（剣士／魔導士／弓兵／暗殺者／騎士／司祭）と役割名（タンク／バッファ／ヒーラー／専守タンク）が混在＝ Name 規則の「役職」相当で曖昧可
- 固有ユニットは UniqueUnitIds 定数で参照

実装一覧の SSoT は実装側 Roster。本仕様書には対応表を持たない（二重ドリフト防止・[900 §8.4](900_development_rules.md)）。

##### 帝国軍命名規則（敵側）

**Name**：「**帝国 + 役職**」の 6 種類で味方コピー兵を被せ＋皇太子戦専用 1 体。

| Name | 構成体数 | 内訳（味方コピー）|
|---|---|---|
| 帝国剣士 | 2 | 火剣士／水剣士 |
| 帝国弓兵 | 2 | 火弓兵／水弓兵 |
| 帝国大盾兵 | 2 | 火タンク／水タンク |
| 帝国補助兵 | 2 | 火バッファ／水バッファ |
| 帝国騎士 | 1 | 光騎士 |
| 帝国司祭 | 1 | 光司祭 |
| 帝国大魔導士 | 1（皇太子戦専用）| 火魔導士 |

**同名異 ID**：プレイヤーから見ると「同じ見た目のユニットが行動だけ違って見える」（例：帝国大盾兵 A は反撃強化／帝国大盾兵 B は専守＝被ダメ -30%）。敵側の戦術バリエーションを保つための意図的設計。

**Element**：敵側は **一律無属性**。VSプロト範囲では地形補正は全列 Neutral 固定でシナジー発動も味方陣営限定なので、敵側は属性を持つ意味がない。能力差（反撃強化／専守等）は永続パッシブ効果で残る。

**Id**：味方の Id に `imperial_` プレフィックスを付与する。

味方のレア相当に当たる帝国軍（暗殺者／専守タンク／ヒーラー／光魔導士）はプロト範囲では持たない（皇太子戦の固定編成で使う火魔導士のみ残置）。

### 1.13 R4 救出戦ショートカット

第三者にゲーム全体を短時間で体験させるための、タイトル画面のショートカットボタン。「あと一手で救出できる手前」状態（R4 開始・左列敵領制圧済・手駒 12 体全員 Lv2）から一時的にラン開始し、救出戦 → R5 → R7 ボス戦・エンディングまで通常進行でプレイできる。

#### 1.13.1 ショートカットセーブ

| セーブ | スタート状態 | プレイ内容 |
|---|---|---|
| 救出戦体験（`Save2`） | R4 開始・左列敵領制圧済・手駒 12 体（王女含め全員 Lv2） | R4 から通常進行（内政＋全戦線戦闘）。左列敵拠点（救出戦）勝利 → B-d ブリジット加入 → R5 続行 → R7 まで一通りプレイ可能 |

USP の核「救うためにもう一度挑みたくなる」のピーク＝救出ピークを軸に、その先 R7 まで通しでプレイ可能にして審査員にゲーム全体像を体験させる。

#### 1.13.2 起動経路

タイトル画面の「**救出戦から始める（R4）**」ボタンを押下すると：

- ラン中の一時状態（手駒・マップ状態・CurrentRound）を `DemoSaveDefinition` のスナップショットから構築
- マップ構築時の `hasRescuedBalduin` は常に false 固定＝既存メタ進行で救出済プレイヤーでも左列バルドゥイン拠点として描画され救出戦を再体験できる
- **メタ進行（PlayerPrefs）は触らない＝既存セーブを破壊しない**（既存メタ強化が乗った状態で R4 開始する）
- エンディング後は通常通り `FinishCurrentRun` で Hub に戻り、メタ進行も通常通り保存される

開始ラウンドを R4 にした理由：R6 開始セーブだと R5 B-b2（バルドゥイン降伏演出）を経由しないまま R6 開始時 B-c（謎の少女）の発火条件に該当して「救援するつもりだったのに少女が踵を返す」破綻が発生する。R4 開始ならどのラウンド開始演出にも該当せず、救援を実行する正規ルートの中間として綺麗にハマる。

| 7 | 永続加入＋マップ動的化 | 次ラン以降 | 初期手駒にブリジット／左列敵拠点のバルドゥイン籠城拠点扱いを解除＝中央列・右列と同じ通常の敵拠点に（[330 §7](330_vsprototype_storyplot.md)） |

### 2.2 加入条件の段階構造

左列の敵拠点（バルドゥインが籠城する特殊拠点）の攻略には左列で順に勝つ必要がある：

```
R1: 左列敵領で勝利（最速）→ 制圧
R2: 左列敵拠点で勝利 → ブリジット加入確定（最速到達ルート）

R5 までに左列敵拠点を制圧できなければ加入不可（=救援失敗ルート）
```

R3-R4 への遅延も許容（プレイヤーが他列の防衛で手一杯のケース）。

### 2.3 ブリジット・王女の戦闘性能

詳細仕様（基本ステ・攻撃技・固有パッシブ／Waza・強化選択肢）は [320 §1.2 / §4.9](320_vsprototype_combat_spec.md) ＋実装側 Roster を SSoT とする。本書には「ブリジットは火属性・近接アタッカー／王女は光属性・近接アタッカーで全体オーラ持ち」という配役だけ示す。

### 2.4 ストーリー素材

ストーリーテキスト・スチル指定は [330 §2〜§3](330_vsprototype_storyplot.md) を参照。

## 3. メタ通貨「王国の記憶」

### 3.1 獲得式

獲得式と想定獲得量は検討中。現 SSoT は実装側のメタ報酬式定義。

### 3.2 消費先

| 強化項目 | コスト（段階別） | 効果 | 上限 |
|---|---|---|---|
| **王女 Lv 強化** | 30 → 60 | ラン開始時の王女初期 Lv +1（購入時に §1.11.5 の 3 択から選択） | 2 回まで（Lv3 上限） |
| **ブリジット Lv 強化** | 30 → 60 | ラン開始時のブリジット初期 Lv +1（購入時に §1.11.5 の 3 択から選択・R5 までに加入していないと無意味） | 2 回まで（Lv3 上限） |
| **行動力 +1**（毎ラウンドの内政枠 2→3） | 100 | 永続 | 1 回まで |
| **初期所持ユニット +1**（初期ドラフト枠 +1） | 50 → 80 → 100 | 永続。**ランダム自動ではなくドラフト 3 択でプレイヤーが選ぶ** | 3 回まで |

**コスト構造**：各強化は段階別コストを保持し、現 Lv に応じて次段階購入コストを返す形で残高消費・残高返戻 API を提供する。

**固有ユニット Lv 強化の動作**：メタ強化画面で「王女 Lv 強化」を購入すると 3 択モーダルが開き、王女の強化選択肢から 1 つ選ぶ。選択結果は永続化され、次ラン開始時にそのランの王女に順次適用される。

**バランス曲線の意図**：初期は「防衛だけで精一杯」の設計。メタ強化で初期ユニット数・行動力を増やすことで防衛に余裕が生まれ、反転攻勢→バルドゥイン救出に手が伸びるようになる。「メタ強化＝間接強化（上限解放／初期値底上げ）」「ラン内＝直接強化（召集／ユニット強化）」の二層構造を実現する。

### 3.3 永続化

- 保存先：永続セーブストア（プロト範囲では PlayerPrefs を採用・キー `vsproto_meta_progress`）
- 保持内容：メタ通貨残高／完了ラン数／永続加入権リスト／適用済み強化（ユニット ID → 購入順 Upgrade ID リスト）／トゥルーエンド到達フラグ／既見ストーリーシーン ID 集合
- スキーマの詳細は実装側 SaveStore 実装を SSoT とする

## 4. メタ拠点 UI（亡国の遺跡）

### 4.1 名称・位置づけ

- VSプロトでの呼称は **「メタ拠点＝亡国の遺跡」**
- ラン外で表示される永続強化画面。タイトル → メタ拠点 → ラン開始 → ラン終了 → メタ拠点 のループ

### 4.2 機能要件

- **上部**：メタ通貨残高・周回数・トゥルーエンド到達フラグ表示
- **中央**：強化選択リスト（§3.2 の 4 項目を上から順に「行動力 +1」「初期所持ユニット +1」「王女 Lv 強化」「ブリジット Lv 強化」）
  - 各項目に「現 Lv に応じた次段階コスト」「効果」「現在 Lv」表示
  - 残高不足の項目はグレーアウト
  - 上限到達時はコスト表示を「-」＋ボタンを「上限到達」に
  - **ブリジット Lv 強化は未解禁時は専用の「ブリジット : 未解禁」テキスト 1 行に差し替え**（コスト・購入ボタン非表示／**解禁条件はメニュー本文に書かずゲーム内チュートリアル or ゲーム外ガイドで伝える方針**）
- **最下部**：「次のランへ →」ボタン

#### 背景画像

- 背景画像をフル画面で ScaleAndCrop 描画。アセット未配置時は単色塗りフォールバック
- 中央パネルは半透明（α=0.88 仮置き・うっすら背景が透ける程度）にして拠点の雰囲気を出す

### 4.3 タイトル → メタ拠点遷移

タイトル「ゲームスタート」ボタンから以下に分岐：

- セーブあり → メタ拠点 → ラン → エンディング → メタ拠点戻り
- セーブなし（初回）→ A-a プロローグ → 固定構成スタート（メタ拠点経由なし）

初回プレイ判定はセーブファイル存在ベース（完了ラン数 0 ではなく、永続化の有無で判定）。

タイトル画面の構成：

| 位置 | 要素 | 役割 |
|---|---|---|
| 中央メイン | ゲームスタート（260×60） | 通常プレイ動線（上記分岐） |
| 中央直下 | ゲーム終了（220×44・一回り小さく） | `Application.Quit`（Editor では再生停止） |
| 右下隅 | DEMO: 救出戦から始める(R4)（300×38・控えめ配色） | R4 ショートカット（§1.13）・あくまでデモ用と分かるよう右下端に控えめ配置 |

**背景画像にロゴが含まれる前提**のため、画像が配置されている場合はゲーム名テキストを描画しない（未配置時のみ単色塗り＋フォールバックでゲーム名「Echoes of the Lost Kingdom」を中央表示）。

### 4.4 オミット範囲

- 拠点ビジュアル進化（廃墟→当時の姿）：試遊版送り
- 3 施設ツリー（訓練所・図書館・神殿）：試遊版送り
- 英霊との会話：試遊版送り
- 固有キャラの拠点登場演出：試遊版送り
